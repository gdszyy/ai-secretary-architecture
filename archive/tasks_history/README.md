# 历史任务归档说明

`archive/tasks_history/` 用于保存与 AI 秘书核心架构演进相关的历史任务材料，目的是保留决策来源、审核痕迹和最终文档副本，便于后续审计与人工追溯。

## 默认读取规则

> 该目录**默认不是 Agent 必读上下文**。

除非任务明确要求追溯历史来源、核对任务原件或分析某一设计演进路径，否则不建议在新任务中主动读取本目录。

## 当前归档策略

| 保留内容 | 说明 |
|------|------|
| `README.md` | 保留任务摘要与交付背景 |
| `review.md` | 保留审核意见与驳回记录 |
| `deliverables/*.md` / `deliverables/*.txt` | 保留最终 Markdown / 文本交付物 |

| 排除内容 | 说明 |
|------|------|
| `deliverables/*.json` | 原始数据量大，且不适合作为主上下文 |
| `deliverables/*.py`、`*.ts`、`*.tsx` | 属于实现或处理中间产物，不纳入本仓库主档案 |
| 大型前端代码目录 | 属于其他仓库的候选内容，不保留在本仓库主上下文 |

## 使用建议

当需要验证某份正式文档的来源时，应优先查看 `docs/` 下的正式版本，再回到此目录查找对应任务的历史记录，而不是反向以归档版本作为当前事实来源。
