# XPBET 首页推荐与活动功能 CMS 配置说明文档

**版本**：1.1
**作者**：Manus AI
**模块**：首页推荐系统、活动系统

本文档旨在为运营人员提供 XPBET 首页推荐与活动功能 CMS 后台的配置操作指南，详细说明各配置项的业务含义及其与前端展示的映射关系。

---

## 1. 首页推荐系统配置

### 1.1 热门比赛/联赛配置

**业务含义**：手动指定在首页推荐区域展示的热门比赛或联赛。当前为简化版，不依赖复杂算法，完全由运营人员手动控制。

**配置项说明**：

| 配置项 | 业务含义与操作指南 |
| :--- | :--- |
| **推荐类型** | 选择是推荐单场“比赛”还是整个“联赛”。 |
| **运动类型** | 选择对应的运动（如足球、篮球），用于前端分类展示。 |
| **赛事/联赛 ID** | 输入底层数据平台对应的唯一标识符（如 `evt_1234567`）。 |
| **推荐名称** | 前端展示的标题（如“英超 — 曼城 vs 阿森纳”）。 |
| **展示权重** | 1-100 的整数。权重越高的推荐项在前端排序越靠前。 |
| **有效期至** | 超过此时间后，该推荐项将自动下线。不填则长期有效。 |
| **状态** | 控制该推荐项是否立即在前端生效。 |

**前端映射**：配置生效后，将在首页的“热门推荐”横向滑动卡片区展示。前端会根据“展示权重”降序排列。

### 1.2 串关推荐卡片配置

**业务含义**：配置首页展示的串关推荐卡片，吸引用户进行串关投注。

**配置项说明**：

| 配置项 | 业务含义与操作指南 |
| :--- | :--- |
| **卡片标题** | 前端卡片顶部展示的标题（如“今日精选 2 串 1”）。 |
| **串关类型** | 选择是几串几（如 2 串 1、3 串 1）。 |
| **展示位置** | 指定该卡片在前端推荐区的具体位置（如位置 1、位置 2）。 |
| **推荐赛事 ID 列表** | 输入需要组合的赛事 ID。**注意**：输入的赛事数量必须与“串关类型”一致（例如 3 串 1 必须输入 3 个赛事 ID）。 |
| **有效期至** | 卡片的展示截止时间。 |

**前端映射**：前端根据配置的赛事 ID 实时拉取最新赔率，自动计算串关总赔率，并渲染为一张完整的串关推荐卡片。用户点击卡片上的“Place Buy”即可一键将这些赛事加入投注单。

### 1.3 中奖订单推荐配置

**业务含义**：在首页展示近期用户的中奖订单，利用从众心理激励其他用户投注。

**配置项说明**：

| 配置项 | 业务含义与操作指南 |
| :--- | :--- |
| **展示模式** | 当前简化版仅支持“最近中奖订单（时间倒序）”。 |
| **展示数量** | 前端滚动展示的订单总数（最多 20 条）。 |
| **最低中奖金额过滤** | 设置一个门槛（如 $100），低于此金额的订单不会被展示。 |
| **数据时效范围** | 限制只展示最近 24 小时、7 天或 30 天内的订单。 |
| **用户信息脱敏** | 开启后，前端展示的用户名将被打码（如 `J***o`），保护用户隐私。 |
| **展示中奖金额** | 控制是否在前端明文显示具体的中奖金额数字。 |

**前端映射**：配置生效后，首页将出现一个垂直或水平滚动的“中奖播报”模块，实时拉取符合条件的真实中奖数据进行展示。

### 1.4 卡片排序管理

**业务含义**：全局控制首页各个推荐模块（如热门比赛、串关卡片、活动 Banner 等）的上下排列顺序。

**配置操作**：在后台列表中，通过鼠标拖拽行首的拖拽手柄来调整模块的上下顺序。调整完成后，必须点击右上角的“发布排序”按钮，新的顺序才会同步到前端。此外，可以通过右侧的开关单独控制某个模块的显示或隐藏。

**前端映射**：前端首页的整体页面结构将严格按照此处的排序结果进行自上而下的渲染。

## 2. 活动系统配置

### 2.1 Superodds (超级赔率) 活动配置

**业务含义**：为特定赛事或运动提供额外的赔率加成，吸引用户投注。

**配置项说明**：

| 配置项 | 业务含义与操作指南 |
| :--- | :--- |
| **活动名称** | 内部管理及前端活动页展示的名称。 |
| **赔率加成比例** | 在原始赔率基础上增加的百分比（如填 10，表示加成 10%）。 |
| **适用运动 / 赛事 ID** | 限定该活动只对特定运动（如足球）或特定几场比赛生效。 |
| **最低投注金额** | 用户单笔投注必须达到此金额才能享受超级赔率。 |
| **最大加赔金额上限** | 限制平台风险，单笔投注最多能获得的额外加成金额。 |

**前端映射**：符合条件的赛事，其赔率按钮上会出现特殊的“Superodds”高亮标识（如闪电图标或特殊颜色）。用户将该赔率加入投注单后，投注单内会明确展示原始赔率与加成后的超级赔率对比。

### 2.2 串关加赔活动配置

**业务含义**：鼓励用户进行多场次的串关投注。串关数量越多，平台给予的额外赔率加成比例越高。

**配置项说明**：

| 配置模块 | 业务含义与操作指南 |
| :--- | :--- |
| **基本信息** | 配置活动名称、起止时间及前端展示的活动描述。 |
| **加赔比例** | 配置阶梯式的加赔规则。例如：2 串 1 加赔 5%，3 串 1 加赔 8%，4 串 1 加赔 12%。 |
| **加赔上限设置** | 设置单笔最大加赔金额和每用户每日加赔上限，用于控制平台成本。 |
| **参与条件** | 可限制最低或最高投注金额、用户 VIP 等级或是否完成 KYC 验证。 |
| **适用范围** | 可排除特定赛事（如已经参与了 Superodds 的赛事），或限定只对“全场胜平负”等特定盘口生效。 |

**前端映射**：当用户在投注单中选择串关投注时，系统会根据当前串关数量自动匹配对应的加赔比例。投注单底部会清晰展示当前享受的额外加赔比例，并实时计算出加赔后的可赢金额。

### 2.3 营销入口配置

**业务含义**：统一管理各类活动（如 Superodds、串关加赔、每日特惠等）在前端的入口展示。

**配置项说明**：

| 配置项 | 业务含义与操作指南 |
| :--- | :--- |
| **入口名称** | 内部管理名称。 |
| **入口类型** | 选择是活动 Banner、导航图标还是弹窗。 |
| **展示位置** | 指定该入口出现在首页顶部、推荐区还是底部导航栏。 |
| **跳转链接** | 用户点击该入口后跳转的页面路径（如 `/promotions/superodds`）。 |
| **展示优先级** | 当同一位置有多个入口时，数字越小（优先级越高）的入口排在越前面。 |
| **展示状态** | 一键开启或关闭该入口。 |

**前端映射**：前端各个预设的广告位会根据此配置动态拉取并渲染对应的入口图片或按钮。点击后执行配置的跳转逻辑。
