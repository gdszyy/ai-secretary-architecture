# 体育博彩平台 APP 移动端设计规范（面向 AI Agent）

**版本**：1.0
**作者**：Manus AI
**目标受众**：负责生成、设计或重构移动端（H5/原生 APP）前端代码的 AI Agent

## 1. 概述

本文档为 AI Agent 提供一套统一、标准化的体育博彩平台 APP 移动端设计规范。本规范基于 Web 端亮色主题设计规范（v1.2）扩展而来，继承了 Web 端的颜色、圆角等基础 Design Token，并针对移动端设备的物理特性、触控交互和屏幕空间进行了专属适配。

AI Agent 在生成或修改移动端代码时，**必须**严格遵循本规范中的 Token 差异和布局规则，以确保移动端体验的流畅性、易用性和一致性。

---

## 2. 移动端 Design Token 差异

移动端与 Web 端共享同一套颜色（Color）和圆角（Radius）系统，但在字体、间距和安全区处理上存在显著差异。AI Agent 必须应用以下覆盖规则。

### 2.1 字体尺寸缩放 (Typography Scaling)

移动端屏幕较小，需对 Web 端字体进行等比缩放，确保阅读舒适度。全站统一使用 **Poppins** 字体。

| 级别 | 移动端字号 | Tailwind | 字重 | 用途 |
| :--- | :--- | :--- | :--- | :--- |
| 辅助文字 | 10px | `text-[10px]` | Light 300 | 底部 Tab 栏文字、极次要说明 |
| 辅助说明 | 12px | `text-xs` | Normal 400 | 快捷按钮、普通辅助说明 |
| 正文内容 | 14px | `text-sm` | Normal 400 | 主要内容文字、普通标签 |
| 标签名称 | 14px | `text-sm` | Medium 500 | 盘口名称、分类标签 |
| 赔率数字 | 16px | `text-base` | Bold 700 | 赔率数字、联赛名称、关键标题（较 Web 端放大） |
| 金额输入 | 14px | `text-sm` | SemiBold 600 | 金额输入框内的货币符号和数字 |

### 2.2 触控目标与间距 (Touch Target & Spacing)

移动端依赖手指触控，必须满足各大平台的最小触控目标要求。

- **iOS 标准**：最小触控区域为 44×44pt。
- **Android 标准**：最小触控区域为 48×48dp。
- **统一规范**：所有独立可点击元素（图标按钮、单选框等）的触控区域**必须**达到 `48×48px`（`w-12 h-12`）。当视觉尺寸较小时，使用 `padding` 扩展触控区域。

```css
/* 示例：视觉 24px，触控区域 48×48px */
.mobile-icon-btn {
  width: 24px;
  height: 24px;
  padding: 12px;
  box-sizing: content-box;
}
```

### 2.3 安全区与状态栏 (Safe Area & Status Bar)

移动端设备存在刘海屏、打孔屏和底部小白条，必须使用 CSS 环境变量适配安全区。

- **状态栏高度**：顶部导航栏必须预留状态栏高度，使用 `pt-[env(safe-area-inset-top)]`。
- **底部安全区**：吸底元素（如 Tab Bar、BetBar）必须预留底部安全区，使用 `pb-[env(safe-area-inset-bottom)]`。
- **左右安全区**：横屏模式下，使用 `pl-[env(safe-area-inset-left)]` 和 `pr-[env(safe-area-inset-right)]`。

```html
<!-- 示例：适配安全区的吸底 Tab Bar -->
<div class="fixed bottom-0 left-0 right-0 bg-white pb-[env(safe-area-inset-bottom)]">
  <div class="h-14 flex items-center justify-around">...</div>
</div>
```

## 3. 移动端布局范式

### 3.1 底部导航栏 (Tab Bar)

- **结构**：固定在屏幕底部（`fixed bottom-0`），高度 `h-14` (56px)，包含 3-5 个核心导航项。
- **样式**：背景白色，顶部边框 `border-t border-ft-c`，图标尺寸 24×24px，文字字号 10px。
- **状态**：选中态图标和文字使用品牌红 `#E80104`，未选中态使用次要文字色 `ft-e`。

### 3.2 全屏滑动与吸顶/吸底元素

- **全屏滑动**：主内容区域使用 `flex-1 overflow-y-auto`，隐藏原生滚动条（`scrollbar-width: none`）。
- **吸顶元素 (Sticky Header)**：分类导航、联赛筛选等使用 `sticky top-0 z-10 bg-white`，确保滚动时始终可见。
- **吸底元素 (Fixed Bottom)**：BetBar、悬浮操作按钮（FAB）使用 `fixed bottom-0 z-20`，并叠加安全区适配。

### 3.3 抽屉面板 (Drawer)

- **用途**：用于展示侧边导航、筛选面板或复杂的设置项。
- **布局**：从屏幕左侧或右侧滑出，宽度占屏幕的 80%（`w-[80vw]`），剩余 20% 为半透明遮罩（`bg-black/40`），点击遮罩关闭抽屉。

## 4. 移动端核心组件适配

### 4.1 底部投注栏 (BetBar) → 全屏抽屉

Web 端的 BetBar 在移动端需适配为全屏抽屉，以提供充足的操作空间。

- **收起态**：悬浮于 Tab Bar 上方（或页面底部），高度 `h-14`，显示已选数量和「展开」按钮。
- **展开态**：点击后从底部向上滑出，占据全屏（或 90% 屏幕高度），包含完整的投注单（Betslip）功能。
- **关闭操作**：提供明确的下拉把手（Handle）和右上角关闭按钮（48×48px 触控区）。

### 4.2 赛事卡片 → 竖向滚动列表

Web 端的网格布局在移动端改为单列竖向滚动列表。

- **容器**：宽度 `w-full`，左右内边距 `px-4`，卡片间距 `gap-3`。
- **信息排布**：队伍名称和比分垂直排列，赔率按钮组放置在卡片底部，占据整行。
- **赔率按钮**：高度 `h-12` (48px)，等宽分布（`flex-1`），确保单手拇指轻松点击。

### 4.3 盘口按钮 → 大尺寸触控变体

- **尺寸**：高度从 Web 端的 40px 提升至 `h-12` (48px)。
- **排版**：盘口名称和赔率数字在按钮内垂直居中，两端对齐（`flex justify-between items-center px-4`）。

### 4.4 投注单 (Betslip) → 底部 Sheet 面板

- **触发**：点击具体赔率后，从底部弹出 Sheet 面板。
- **高度**：根据内容自适应高度，最高不超过屏幕的 80%。
- **金额输入**：唤起数字键盘时，Sheet 面板必须自动上移，避免被键盘遮挡。

## 5. 移动端手势交互规范

移动端强依赖手势操作，需提供符合直觉的交互反馈。

- **下拉刷新 (Pull-to-Refresh)**：在列表顶部下拉，触发数据刷新。需显示品牌红色的加载指示器（Spinner）。
- **左滑删除 (Swipe-to-Delete)**：在投注单列表中，对单个投注项向左滑动，露出红色的删除按钮（`bg-lost text-white`），点击或继续滑动完成删除。
- **长按 (Long Press)**：长按赛事卡片，弹出快捷操作菜单（如收藏、分享）。触发时间为 500ms，伴随轻微震动反馈（Haptic Feedback）。
- **双击 (Double Tap)**：双击视频直播区域，切换全屏/退出全屏。

## 6. 移动端动效规范

动效需遵循物理规律，提供流畅的视觉引导。

### 6.1 页面转场 (Transitions)

- **Push (层级深入)**：新页面从右侧滑入（`translate-x-full` → `translate-x-0`），旧页面向左微移并降低透明度。
- **Modal (模态弹出)**：从底部滑入（`translate-y-full` → `translate-y-0`），用于 BetBar 展开、筛选面板。
- **Fade (渐隐渐现)**：用于 Tab 切换、Toast 提示，透明度 `opacity-0` → `opacity-100`。

### 6.2 时长与缓动曲线 (Duration & Easing)

- **微交互（按钮点击、状态切换）**：时长 150ms，曲线 `ease-out`。
- **面板展开/收起**：时长 300ms，曲线 `cubic-bezier(0.2, 0.8, 0.2, 1)`（快速启动，平滑减速）。
- **页面转场**：时长 400ms，曲线 `cubic-bezier(0.3, 0, 0.2, 1)`。

```css
/* Tailwind 动效配置示例 */
theme: {
  extend: {
    transitionTimingFunction: {
      'panel': 'cubic-bezier(0.2, 0.8, 0.2, 1)',
      'page': 'cubic-bezier(0.3, 0, 0.2, 1)',
    },
    transitionDuration: {
      '150': '150ms',
      '300': '300ms',
      '400': '400ms',
    }
  }
}
```

## 7. 拇指热区与键盘适配

### 7.1 拇指热区分布 (Thumb Zone)

设计需对单手操作友好，核心操作必须放置在屏幕中下部的「舒适区」。

- **舒适区（屏幕下半部）**：放置高频操作，如 Tab Bar、BetBar、主要 CTA 按钮（Place Buy）。
- **伸展区（屏幕中上部）**：放置次频操作，如赛事卡片、盘口选择。
- **困难区（屏幕顶部）**：放置低频操作，如返回按钮、全局设置、个人中心入口。

### 7.2 键盘弹出适配 (Keyboard Avoidance)

当用户在输入框（如投注金额）唤起软键盘时，界面必须进行适配：

- **输入框可见性**：当前获焦的输入框必须自动滚动到键盘上方，保持完全可见。
- **吸底元素隐藏**：键盘弹出时，隐藏非必要的吸底元素（如 Tab Bar），为内容腾出空间。
- **完成按钮附加**：在键盘顶部附加一个「完成」或「确认」工具栏（Toolbar），方便用户快速收起键盘并提交。

## 9. 国际化 (i18n) 专项规范：葡萄牙语与西班牙语

针对拉美（如巴西）及欧洲（如葡萄牙、西班牙）市场，APP 的设计与开发必须满足葡萄牙语（pt-PT, pt-BR）和西班牙语（es-ES, es-MX）的本地化排版与格式要求。

### 9.1 文本扩展率 (Text Expansion)

葡萄牙语和西班牙语的文本长度通常比英语长 **15% 到 30%**。在移动端有限的屏幕空间内，极易出现文本截断或换行问题。

- **弹性布局**：严禁为包含文本的按钮、标签或卡片设置固定宽度（Fixed Width）。必须使用弹性布局（如 `flex-1`, `w-full`, `max-w-full`）和自适应高度。
- **多行支持**：对于长文本（如盘口名称、赛事说明），必须允许换行（`whitespace-normal`），或在空间极度受限时使用优雅的文本截断（`truncate`），但需确保核心信息可见。
- **按钮设计**：CTA 按钮（如「确认下注」）应预留充足的水平内边距（如 `px-6`），以容纳翻译后变长的词汇（例如 English: "Bet" -> Portuguese: "Apostar" -> Spanish: "Apostar"）。

### 9.2 变音符号与字体选型 (Diacritics & Typography)

葡萄牙语和西班牙语包含大量带有变音符号的字符（如 á, é, í, ó, ú, ñ, ç, ã, õ 等）。

- **字体支持**：全站统一使用的 **Poppins** 字体原生支持拉丁语系扩展（Latin Extended），能够完美渲染西语和葡语的变音符号，无需替换字体。
- **行高调整**：带有变音符号的大写字母（如 Á, Ñ）可能会与上一行的下行字母（如 p, y, g）发生重叠。移动端正文行高（Line Height）必须至少保持在 `1.5`（Tailwind 中的 `leading-normal` 或 `leading-relaxed`），标题行高不低于 `1.2`（`leading-tight`）。

### 9.3 数字与货币格式 (Number & Currency Formatting)

这是博彩 APP 本地化最核心的环节，直接影响用户的资金认知和信任度。

- **小数点与千位分隔符**：
  - **葡萄牙语 (pt-PT/pt-BR)** 和 **西班牙语 (es-ES)** 均使用**逗号 (`,`)** 作为小数点，使用**点 (`.`)** 或**空格**作为千位分隔符。
  - 示例：一百万点五十，格式化为 `1.000.000,50` 或 `1 000 000,50`（严禁使用美式的 `1,000,000.50`）。
- **货币符号位置**：
  - **欧元 (€)**：在葡萄牙和西班牙，货币符号通常位于数字**之后**，并带有一个空格（例如：`150,00 €`）。
  - **巴西雷亚尔 (R$)**：在巴西，货币符号通常位于数字**之前**，并带有一个空格（例如：`R$ 150,00`）。
- **代码实现**：前端必须使用 `Intl.NumberFormat` 进行动态格式化，严禁硬编码分隔符。
  ```javascript
  // 葡萄牙语（葡萄牙）欧元示例
  new Intl.NumberFormat('pt-PT', { style: 'currency', currency: 'EUR' }).format(1234.56); 
  // 输出: "1 234,56 €"

  // 葡萄牙语（巴西）雷亚尔示例
  new Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(1234.56); 
  // 输出: "R$ 1.234,56"
  ```

### 9.4 日期与时间格式 (Date & Time Formatting)

- **日期格式**：西语和葡语国家普遍使用 **DD/MM/YYYY** 格式（如 `28/03/2026`），严禁使用美式的 MM/DD/YYYY。
- **时间格式**：普遍使用 **24 小时制**（如 `14:30`），不使用 AM/PM 后缀。赛事时间的展示必须严格遵循此规范。
