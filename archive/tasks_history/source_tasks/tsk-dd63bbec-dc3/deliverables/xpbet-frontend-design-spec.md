# XPBET 首页推荐与活动功能前端设计规范

**版本**：1.0
**作者**：Manus AI
**模块**：首页推荐系统、活动系统（Superodds、串关加赔）

## 1. 概述

本文档旨在为 XPBET 体育博彩平台的首页推荐区域及核心营销活动（Superodds、串关加赔）提供详细的前端设计规范。本规范基于 XPBET 全局深色主题（Dark Theme）设计语言，明确了颜色、字体、间距、圆角等基础视觉元素的标准值，并对新增的核心组件结构进行了规范。设计规范的严格执行是确保全站视觉一致性和提升用户体验的基础。

## 2. 设计 Token (Design Tokens)

### 2.1 颜色系统 (Color System)

在深色主题下，颜色系统以深蓝与深灰为底色，辅以品牌红作为主操作色，并引入金色和绿色作为特定业务场景的强调色。这种色彩搭配不仅符合体育博彩平台的行业惯例，还能有效引导用户的视觉焦点。

**基础背景与表面色 (Background & Surface)**

| Token | 颜色值 | 用途说明 |
| :--- | :--- | :--- |
| `--dark-bg` | `#0A0E1A` | 页面最底层背景色，提供深邃的视觉基础。 |
| `--dark-surface` | `#161B22` | 顶部导航栏、底部 Tab 栏以及各类浮层的背景色。 |
| `--dark-card` | `#1C2333` | 赛事卡片、推荐卡片等内容容器的默认背景色。 |
| `--dark-card-hover` | `#21293A` | 卡片在用户悬停（Hover）状态下的背景色，用于提供交互反馈。 |

**边框与分割线 (Border & Divider)**

| Token | 颜色值 | 用途说明 |
| :--- | :--- | :--- |
| `--dark-border` | `rgba(255,255,255,0.08)` | 常规卡片边框与列表分割线，提供微弱的视觉边界。 |
| `--dark-border-strong` | `rgba(255,255,255,0.15)` | 强调边框以及悬停状态下的边框，增强元素的立体感。 |

**文本色 (Typography Colors)**

| Token | 颜色值 | 用途说明 |
| :--- | :--- | :--- |
| `--dark-text-primary` | `#E6EDF3` | 主要文字、标题以及核心数据（如赔率数值）的颜色。 |
| `--dark-text-secondary` | `#8B949E` | 次要文字与列表辅助信息的颜色，降低视觉权重。 |
| `--dark-text-muted` | `#6E7681` | 极次要文字、时间戳以及输入框占位符的颜色。 |

**品牌与状态强调色 (Brand & Functional)**

| Token | 颜色值 | 用途说明 |
| :--- | :--- | :--- |
| `--primary-0` | `#E80104` | 品牌主色，广泛应用于主按钮、选中状态标识以及 Live 赛事标识。 |
| `--win` | `#27A700` | 成功状态、赔率上涨标识，同时作为**串关加赔活动**的主题色。 |
| `--lost` | `#AF0507` | 失败状态以及赔率下跌标识。 |
| `--gold-1` | `#FFC31D` | 奖励提示、中奖金额展示，同时作为**Superodds 活动**的主题色。 |

### 2.2 字体系统 (Typography)

全站统一使用 **Poppins** 字体，以确保多语言环境下的良好阅读体验。针对移动端优先的布局策略，字号层级经过精心设计，以适应不同屏幕尺寸和信息层级。

| 级别 | 字号 | 字重 | 用途说明 |
| :--- | :--- | :--- | :--- |
| 极小辅助 | 10px | Normal 400 | 底部 Tab 标签以及极次要的说明文字。 |
| 辅助说明 | 11px / 12px | Normal 400 | 赛事时间、联赛名称以及次要状态标签。 |
| 正文内容 | 13px / 14px | Normal 400 | 主要内容文字以及普通操作按钮的文案。 |
| 强调数据 | 14px | Bold 700 | 赔率数字以及常规卡片的标题。 |
| 页面标题 | 16px | Bold 700 | 页面顶部全局标题以及大型卡片的标题。 |
| 巨型数字 | 20px+ | Bold 700 | 中奖金额展示以及总赔率等需要极高视觉权重的数字。 |

### 2.3 圆角与间距 (Radius & Spacing)

圆角和间距系统是构建页面节奏感和模块化布局的关键。合理的间距能够提升信息的易读性，而统一的圆角则增强了界面的亲和力。

**圆角系统**

| Token | 像素值 | 用途说明 |
| :--- | :--- | :--- |
| `--radius-xs` | 4px | 微小标签、状态指示点等微观元素。 |
| `--radius-sm` | 8px | 赛事卡片、赔率按钮以及文本输入框。 |
| `--radius-md` | 16px | 大型推荐卡片以及营销活动 Banner。 |
| `--radius-full` | 999px | 胶囊形状按钮、用户头像以及各类状态徽章。 |

**间距系统**

间距系统分为三个层级。微小间距（4px 或 6px）主要用于图标与文字之间，或者紧凑元素内部的排版。标准间距（8px 或 12px）广泛应用于卡片内部元素的排版，确保信息区块的独立性。区块间距（16px 或 24px）则用于页面不同大模块之间的划分，构建清晰的页面结构。

## 3. 核心组件设计规范

### 3.1 营销活动 Banner (Promo Banner)

营销活动 Banner 用于首页顶部的活动入口展示，采用横向滚动（Carousel）布局以节省垂直空间。其标准尺寸设定为宽度 `320px`，高度 `120px`，并应用 `16px` 的圆角。

对于 Superodds 主题，Banner 背景采用深金色渐变 `linear-gradient(135deg, #1A1200 0%, #3D2E00 40%, #6B4E00 100%)`，边框使用金色 `rgba(255,195,29,0.3)`，并在右侧点缀大型半透明的 ⭐ 图标以强化主题。对于串关加赔主题，背景则切换为深绿色渐变 `linear-gradient(135deg, #001A0D 0%, #003D1E 40%, #006B35 100%)`，边框使用绿色 `rgba(39,167,0,0.3)`，右侧装饰大型半透明的 🎯 图标。

### 3.2 赛事卡片 (Match Card)

赛事卡片是展示热门比赛的核心组件，内部包含联赛信息、队伍信息和赔率按钮。外层容器使用 `--dark-card` 背景色，配合 `8px` 圆角和 `--dark-border` 边框。

卡片标题栏背景为 `rgba(255,255,255,0.04)`，高度根据内容自适应，主要展示联赛名称和比赛时间。当比赛处于 Live 状态时，时间文字采用 `--primary-0` 品牌红，并附带具有呼吸动画效果的红点标识。赔率按钮组在默认状态下背景为 `rgba(255,255,255,0.06)`，选中状态下背景加深为 `rgba(232,1,4,0.15)` 且边框变为品牌红。特别地，Superodds 状态的赔率按钮采用金色视觉体系，背景为 `rgba(255,195,29,0.12)`，并带有显眼的 `⭐ SUPER` 徽章。

### 3.3 串关推荐卡片 (Parlay Recommendation Card)

串关推荐卡片用于在首页展示编辑精选的串关组合，旨在引导用户进行多场次投注。该卡片整体采用绿色系作为强调色，边框设定为 `rgba(39,167,0,0.2)`。

卡片头部使用绿色渐变背景，突出展示总赔率，数值采用 `--win` 绿色加粗大字号。主体部分为列表项，每行展示一场比赛的精选盘口，左侧配有圆形序号标识以增强条理性。卡片底部为操作区，包含加赔提示徽章（例如 `🎯 +5% Bônus`）以及一个背景色为 `--win` 的一键添加至投注单的 CTA 按钮。

### 3.4 中奖订单推荐卡片 (Winning Order Card)

中奖订单推荐卡片通过横向滚动的方式展示最近的大额中奖订单，其主要目的是增强平台的信任感并刺激用户的投注欲望。卡片宽度固定为 `200px`，高度随内容自适应。

在视觉风格上，卡片顶部带有一条从金色至绿色的渐变高亮线条 `linear-gradient(90deg, var(--gold-1), var(--win))`。内容结构依次为：包含渐变背景头像、匿名用户名和时间的用户信息区；使用 `--gold-1` 金色大字号展示的中奖金额区；以及简述玩法（如“Acumulador 4 seleções”）和总赔率徽章的订单详情区。

### 3.5 底部投注栏 (Bet Bar)

底部投注栏是一个悬浮于页面底部（Tab 栏上方）的交互组件，仅当用户在页面中选择至少一个赔率后才会平滑滑出。

布局上，左侧区域用于横向滚动展示已选赔率的 Chip，右侧为固定的“Apostar”（下注）主操作按钮。已选 Chip 的背景色为 `rgba(232,1,4,0.12)`，文字颜色为 `--primary-0`，并包含一个独立的移除（×）操作区。当用户选择的赛事数量达到串关加赔条件（通常为 ≥2 场）时，主按钮的文案会动态更新，实时显示计算后的总赔率以及触发的串关加赔比例（如 `+5%`）。

## 4. 响应式与多语言适配

为确保平台在不同设备和语言环境下的可用性，前端实现必须遵循以下适配原则。

首先，采用移动端优先的策略，所有组件均基于 `390px` 的基准宽度进行设计，并广泛使用 Flexbox 布局以确保在不同屏幕尺寸下的弹性伸缩。其次，必须进行安全区适配，页面的顶部和底部需使用 CSS 环境变量 `env(safe-area-inset-top)` 和 `env(safe-area-inset-bottom)`，为系统状态栏和底部操作条预留充足的空间。

在多语言适配方面，考虑到葡萄牙语（PT）和西班牙语（ES）显著的文本膨胀特性，所有单行文本容器（如队伍名称、联赛名称）必须应用 `overflow: hidden; white-space: nowrap; text-overflow: ellipsis;` 样式进行安全截断，而多行文本则应使用 `line-clamp` 属性限制行数。最后，为保障移动端的交互体验，所有可点击元素（包括按钮和图标）的实际触控区域应尽量达到 `44x44px` 的标准，对于视觉尺寸较小的元素，必须通过增加 `padding` 的方式来扩展其隐形触控区。
