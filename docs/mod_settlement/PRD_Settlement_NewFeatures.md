# 新增功能及验收记录

> **文档类型**: OTHER  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/docx/TIopd0ArzoNAQExC0b2jEAFDpgd  
> **归属模块**: `mod_settlement`  

## 业务目标

完善订单查询、后台代付重送、钱包明细及代收商务配置，提升系统功能和用户体验

## 功能详情

### 1. 订单查询(Transaction Order) (FP-SETTLEMENT-001)

**功能描述**: 提供订单查询功能，待个人中心新版处理

### 2. 后台代付重送机制 (FP-SETTLEMENT-002)

**功能描述**: 解决验收未通过的代付重送失败问题

**业务规则**:
- 验收未通过，重送没有成功

### 3. 配置平台充提最高与最低 (FP-SETTLEMENT-003)

**功能描述**: 设置平台充值和提现的最高与最低额度

**业务规则**:
- 参考 Setting 的功能

### 4. 后台用户明细/钱包明细 (FP-SETTLEMENT-004)

**功能描述**: 展示用户主钱包及钱包明细，区分主能量和提现流水

**业务规则**:
- 主钱包展示两种状态：Bonus 能量池、提款限额进度
- 钱包明细默认展示总额，点击后展示明细
- 主能量和提现流水栏位区分展示

### 5. 注单详情内的多个 Bonus 钱包投注及返奖金额展示 (FP-SETTLEMENT-005)

**功能描述**: 在注单详情中展示多个 Bonus 钱包的投注金额及返奖金额

**业务规则**:
- 支持多个 Bonus 钱包金额展示

### 6. 代收商务配置调整 (FP-SETTLEMENT-006)

**功能描述**: 支持同一三方金流商多币别代收代付，调整 Merchant ID 与 API Key 可重复

**业务规则**:
- Merchant ID 与 API Key 可重复
- 同一三方金流商支持多币别代收代付
- Pay In / Out Code 不同，调整栏位存放方式
- 通过手机号校验币别
