# Sportradar UOF结算与冲正消息处理规范

> **文档类型**: TECH_SPEC  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/wiki/MP02wIWReispxwk6VEkjkQuzp5p  
> **归属模块**: `mod_settlement`  

## 业务目标

实现平台对Sportradar统一赔率接口(UOF)结算及纠错消息的自动化准确处理，保障资金安全和用户账户准确性。

## 功能详情

### 1. bet_settlement (投注结算) (FP-SETTLEMENT-001)

**功能描述**: 通知系统一个或多个盘口的最终赛果，触发派彩或结算为输。

**业务规则**:
- 优先采纳certainty=2的官方最终结果覆盖certainty=1的实时结果。
- result与void_factor必须组合解读，确定最终派彩行为。
- void_reason提供作废原因代码，辅助客服和数据分析。
- 盘口唯一定位需使用event_id, product, market_id, specifiers四个字段。

### 2. rollback_bet_settlement (回滚投注结算) (FP-SETTLEMENT-002)

**功能描述**: 纠错指令，用于完全撤销之前的一次或多次bet_settlement操作。

**业务规则**:
- 必须精确定位到需要回滚的盘口（event_id, product, market_id, specifiers）。
- 执行与原始结算完全相反的资金操作，扣回已派发奖金或已退还本金。
- 恢复注单至等待结算状态，清除相关结算结果。
- 冲正导致负余额时，需配套业务规则限制提现、充值抵扣等。

### 3. bet_cancel (投注取消) (FP-SETTLEMENT-003)

**功能描述**: 通知系统将特定盘口或特定时间范围内的投注作废，并退还本金。

**业务规则**:
- start_time和end_time定义取消注单的下单时间范围，必须严格比较注单下单时间。
- void_reason提供取消操作的官方原因代码。
- 单注直接作废退本金，复式/系统投注按投注项作废逻辑处理（赔率按1.0计算）。
- 已结算注单在取消范围内需先冲正再退款。

### 4. rollback_bet_cancel (回滚投注取消) (FP-SETTLEMENT-004)

**功能描述**: 纠错指令，用于撤销之前错误的bet_cancel操作。

**业务规则**:
- 精确定位需要回滚的取消操作。
- 执行与原取消操作相反的资金和状态恢复操作。

### 5. 冲正操作 (FP-SETTLEMENT-005)

**功能描述**: 对已完成的错误账务执行相反操作，纠正错误派彩或退款。

**业务规则**:
- 计算冲正金额，精确扣回错误派发的资金。
- 资金扣除、账变记录和注单状态恢复需在同一数据库事务中完成，保证原子性。
- 允许用户账户余额为负，避免强制阻止扣款导致平台损失。
- 负余额时触发功能限制、用户沟通、欠款回收和风险管理等规则。

### 6. 累积投注与系统投注特殊结算 (FP-SETTLEMENT-006)

**功能描述**: 处理串关和系统投注中投注项作废、赢半、输半等特殊结算情况。

**业务规则**:
- 投注项作废时，赔率按1.0计算。
- 投注项赢半时，转换赔率=(原始赔率-1)*0.5+1。
- 投注项输半时，转换赔率=0.5。
- 冲正金额为原始派彩金额与特殊结算派彩金额的差值。
- 系统投注按相同规则应用于拆分的子注单。
