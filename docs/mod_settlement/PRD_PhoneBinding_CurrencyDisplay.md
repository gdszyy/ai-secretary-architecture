# 手机号绑定国家与币别展示规范

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/docx/NHgXdHIduohxKexLi0KjTHqWpIh  
> **归属模块**: `mod_settlement`  

## 业务目标

根据手机号绑定的国家，展示对应的单一币别，确保注册及相关功能的币别一致性。

## 功能详情

### 1. 登入 (FP-SETTLEMENT-001)

**功能描述**: 展示国别区码并验证手机号，支持墨西哥手机号注册登录。

**业务规则**:
- 手机号需带国码验证，如墨西哥区码 +52
- 手机号格式校验正则：/^\+52?[1-9]\d{9}$/
- 非对应国家手机号弹出多语系错误提示

### 2. KYC (FP-SETTLEMENT-002)

**功能描述**: 调整身份验证字段，支持墨西哥身份证号格式校验。

**业务规则**:
- 字段 CPF 改为 ID Number
- 身份证格式校验正则：^[A-Z]{4}\d{6}[HM][A-Z]{5}[A-Z0-9]\d$
- 非对应国家身份证弹出多语系错误提示
- 后续支持多国家，仅提供格式校验规则，KYC流程三方标准化

### 3. 充/提 (FP-SETTLEMENT-003)

**功能描述**: 根据国别区码展示对应币别及可用渠道。

**业务规则**:
- 充值及提款页面币别展示与注册手机号国家一致
- 充提渠道依据国家选择，当前不变更
- 代付 BankCode: STP，account_type: CLABE，银行信息依据前台配置

### 4. 首页钱包 (FP-SETTLEMENT-004)

**功能描述**: 根据国别区码展示对应币别。

**业务规则**:
- 币别展示依注册手机号国家为主

### 5. 钱包详情 (FP-SETTLEMENT-005)

**功能描述**: 根据国别区码展示对应币别。

**业务规则**:
- 币别展示依注册手机号国家为主

### 6. 责任博彩 (FP-SETTLEMENT-006)

**功能描述**: 根据国别区码展示对应币别。

**业务规则**:
- 币别展示依注册手机号国家为主

### 7. 投注 (FP-SETTLEMENT-007)

**功能描述**: 投注各阶段注单币别展示与用户所选币别一致。

**业务规则**:
- 送出前注单币别与用户选择一致
- 待结算注单币别与用户选择一致
- 已结算注单币别与用户选择一致

### 8. 语系选择 (FP-SETTLEMENT-008)

**功能描述**: 切换语系时币别符号不变，语系变更影响界面语言。

**业务规则**:
- 币别符号固定，依注册帐号币别为主
- 语系切换只影响界面语言，不影响币别及数字展示

### 9. 币别符号 (FP-SETTLEMENT-009)

**功能描述**: 定义墨西哥币与巴西币的符号展示。

**业务规则**:
- 墨西哥币符号为 MAX($)
- 巴西币符号为 BRL(R$)

### 10. 充提功能调整 (FP-SETTLEMENT-010)

**功能描述**: 调整查询条件及界面展示，支持订单号模糊查询。

**业务规则**:
- 查询条件 Currency Type 改为 Currency
- 增加 Order No 查询栏位，支持模糊查找
- 编辑界面加入 Currency 展示

### 11. 帐户信息 (FP-SETTLEMENT-011)

**功能描述**: 统一调整 CPF Number 字段为 ID Number。

**业务规则**:
- 用户列表、用户明细、KYC列表均调整字段名称
