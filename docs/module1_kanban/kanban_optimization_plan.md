# AI Management 看板系统优化与信息源架构设计方案

**作者**: Manus AI  
**日期**: 2026-04-08  
**目标**: 针对当前项目看板的信息断层、自动化收集能力不足及多任务混杂等痛点，构建端到端的信息源追踪、Lark 智能监控与结构化拆分体系。

---

## 1. 痛点分析与优化方向

当前项目看板（Kanban）虽然实现了模块、负责人及里程碑的可视化展示，但其核心数据主要依赖人工总结与被动录入，存在以下关键问题：

1. **信息来源缺失**: 看板数据缺乏可追溯的底层事实支撑（如会议纪要、聊天记录、PRD 变更等）。
2. **多对话混杂难辨**: Lark 等协作工具中，不同团队、不同项目的讨论往往混杂在同一群聊中，难以自动剥离和归因。
3. **里程碑与全局规划脱节**: 缺乏从“大版本规划 -> 模块迭代 -> 阶段里程碑 -> 具体任务”的自上而下的追踪链条。

为解决上述问题，本方案将引入“信息捕获层”与“智能解析层”，将项目管理模式从“被动汇报”升级为“主动感知与智能聚合”。

## 2. 核心信息源体系架构

信息来源必须实现多模态（文字、音频、视频、文档）的统一接入，并经过严格的清洗与验证。

### 2.1 会议记录与多模态流转机制

会议是项目管理中最高密度的信息源。需要构建标准化的会议记录转化流程：

* **采集与转换**: 接入飞书妙记或第三方音视频转录工具，将会议音频/视频转化为结构化文本。
* **智能校验与验证**:
  * **背景比对**: 将转录文本与项目当前的 PRD、架构图及历史看板状态进行交叉比对，识别矛盾点。
  * **完整度评估**: 根据《信息缓冲区机制设计方案》中的评分标准，对提取的行动项（Action Items）进行打分。缺失责任人、截止时间或验收标准的任务，自动触发补充询问。
* **状态映射**: 验证通过的信息自动转化为看板中的子任务或里程碑更新建议。

### 2.2 Lark 机器人监控与智能拆分

Lark 群聊是项目沟通的主阵地。开发专属 Lark 机器人（AI Secretary Bot），实现群聊信息的实时捕获与智能拆分。

* **静默监听与特征提取**: 机器人驻留于核心项目群，实时抓取文本、文件及富文本消息。
* **多对话流分离 (Thread Separation)**:
  * 采用基于上下文和实体的聚类算法，识别并分离同一群聊中并行发生的多组对话。
  * 针对每一组对话，提取核心实体（如：模块名称、讨论对象、特定功能点）。
* **意图识别与子事件拆分**:
  * 将分离后的对话片段输入 LLM，识别其业务意图（如：Bug 报告、需求变更、进度阻塞）。
  * 自动将其拆分为独立的子事件（Sub-events），并匹配至相应的模块和负责人。
* **自动汇总与确认**: 每日/每周生成“群聊工作内容自动汇总”，经项目经理一键确认后，直接同步至看板数据源。

## 3. 模块规划与大版本里程碑体系

里程碑不应是孤立的时间点，而必须是全局规划的具象化体现。

### 3.1 层次化规划模型

构建“大版本 -> 模块 -> 里程碑 -> 任务”的四级追踪体系：

1. **大版本迭代 (Epic / Release)**: 如“XPBET v2025.05 全球站上线”。定义整体商业目标与核心交付物。
2. **模块规划 (Module)**: 如“体育注单系统”、“活动系统”。
3. **里程碑 (Milestone)**: 模块在特定大版本下的关键检查点。例如“Lsport 数据接入完成”、“UOF 生产 Key 验证通过”。
4. **子任务 (Task/Issue)**: 支撑里程碑的具体开发与测试工作，由 Lark 机器人自动抓取或 Meegle 同步而来。

### 3.2 动态里程碑推演

* **数据绑定**: 看板中的里程碑字段必须强制关联至特定的大版本 ID。
* **自动预警**: 当底层子任务的进度出现阻塞（Blocked）或延期时，系统自动向上推演，标记对应的里程碑及大版本存在延期风险，并在看板上以红色预警显示。

## 4. 实施路径与下一步行动

1. **第一阶段 (信息接入)**: 开发 Lark 机器人，实现群聊记录的自动抓取与基础意图识别。
2. **第二阶段 (智能拆分)**: 引入 LLM 多对话分离算法，解决混杂讨论的归因问题，实现自动汇总。
3. **第三阶段 (体系打通)**: 升级 Kanban 数据结构，引入大版本与里程碑的强关联逻辑，并将自动汇总的数据直接渲染至前端看板。

本方案是对现有《AI项目秘书系统落地实施方案》及《信息缓冲区机制设计方案》的深度补充，致力于打通项目管理的“最后一公里”——即隐性知识与碎片信息的结构化捕获。
