# 推荐系统演进路线图（Phase 0–3）

## Phase 0：冷启动（上线第一天可用）

所有模块均由规则引擎驱动，无需任何用户历史数据。输入信号仅包括地理位置、当前时间、全站热度数据和运营人工配置。

**各模块冷启动方案：**

| 模块 | 驱动方式 | 数据来源 |
| :--- | :--- | :--- |
| 比赛推荐 | 全局热度 Top N + 运营置顶 | 全站投注量 + CMS 配置 |
| 超级赔率 | 交易团队手动选品 | CMS 配置 |
| 注单推荐 | 全站最热市场 Top 5 | 全站投注量 |
| 串关推荐 | 人工预制模板库 | CMS 模板库 |
| 热门联赛 | 地区默认静态排序 | CMS 地区配置 |

**扩展预留：** 用户画像服务、个性化推荐引擎、在线学习模块的接口在代码层面预留，暂不激活。

---

## Phase 1：分群个性化

**触发条件：** 积累约 10 万注单 + A/B 测试验证效果。

按运动偏好、投注频率、地区将用户划分为 5–10 个用户群。比赛推荐和热门联赛切换为按群差异化展示，其余三个模块仍保持规则驱动。

**新增工程能力：** 用户分群服务、群体画像存储、A/B 测试框架。

---

## Phase 2：协同过滤

**触发条件：** 行为数据密度足够（约 100 万注单）+ 离线评估指标达标。

引入协同过滤模型和内容相似度模型，批量训练，每日更新。注单推荐和串关推荐切换为模型驱动，超级赔率仍保留人工选品。

**新增工程能力：** 离线训练管道、特征工程服务、模型服务化部署。

---

## Phase 3：实时全个性化

**触发条件：** 实时数据管道就绪 + 在线学习框架部署完成。

引入在线学习模型和实时特征工程，秒级更新推荐结果。5 个模块全部个性化，超级赔率由算法提名候选 + 人工最终确认。

**新增工程能力：** 实时特征流、在线学习框架、实时推理服务。

---

## 始终保留的人工职能

| 职能 | 负责团队 | 原因 |
| :--- | :--- | :--- |
| CMS 运营置顶与活动覆盖 | 内容运营团队 | 商业合作、重大赛事需要人工决策 |
| 超级赔率最终选品 | 交易团队 | 风险敞口需要人工把控 |
| 极端风险审核 | 风险管理团队 | 异常投注模式需要人工判断 |
