# 当周问题当周解决：话题生命周期与纠偏机制设计

## 1. 核心理念

为了避免历史话题无限堆积，并确保项目记录的准确性，引入“当周问题，当周解决”的生命周期管理机制。
核心原则：
1. **时间边界**：所有未闭环话题的生命周期仅限当周，跨周默认视为已解决或已失效。
2. **集中提醒**：每周四在群内集中播报当周遗留问题，促成当周闭环。
3. **人工纠偏**：周报生成后，提供明确的入口供项目负责人（VoidZ）补充或纠正 AI 提取的信息。

---

## 2. 机制一：当周话题自动过期归档

**痛点**：过去的话题如果过了那周，大概率已经通过其他渠道（如 Meegle）解决，继续跟进毫无意义，只会导致看板堆积。

**设计方案**：
- **触发时机**：每周一凌晨（或随周报生成脚本同步触发）。
- **执行逻辑**：
  1. 扫描 Bitable 话题表中所有状态为 `major_decision`、`milestone_fact`、`risk_blocker` 且 `来源周期` 早于当前周的记录。
  2. 将这些记录的状态统一更新为 `已归档(跨周)`。
  3. 在 `追问回复` 字段追加备注：“系统自动归档：跨周默认已解决/失效”。
- **实现载体**：新增脚本 `scripts/topic_expiry_archiver.py`。

---

## 3. 机制二：每周四集中提醒机制

**痛点**：日常频繁追问会打扰群聊，但不追问又容易导致问题遗漏。

**设计方案**：
- **触发时机**：每周四下午 15:00（通过 Manus 定时任务触发）。
- **执行逻辑**：
  1. 扫描 Bitable 话题表中本周新增的、状态为 `risk_blocker`（风险/阻塞）的话题。
  2. 汇总这些未解决的风险项，生成一份 Markdown 清单。
  3. 调用 `lark-secretary` 技能，在指定的项目管理群（如“小组长群”或“GoToBet项目沟通群”）发送富文本卡片。
- **卡片内容示例**：
  - **标题**：🚨 本周遗留风险/阻塞项集中提醒
  - **正文**：列出风险标题、核心事实、所属模块。
  - **行动号召**：请相关负责人确认进度，争取当周闭环。
- **实现载体**：新增脚本 `scripts/weekly_issue_reminder.py`。

---

## 4. 机制三：信息纠正与补充入口

**痛点**：AI 提取的话题可能存在偏差（如将“不接入 Smartico”错误理解，或遗漏了群外的口头决策），需要一个机制让负责人纠正。

**设计方案**：
- **入口位置**：在每周生成的周报卡片（由 `run_weekly_report.py` 发送）底部，增加明确的纠偏指引。
- **纠偏方式**：
  - **方式 A（直接回复）**：负责人在飞书群内直接回复周报卡片，格式如：“@AI秘书 纠正：CRM 系统最终决定接入 Smartico”。AI 秘书读取到该格式后，自动更新 Bitable 记录。
  - **方式 B（文档批注）**：在生成的 Markdown 周报文档中预留 `[人工纠偏区]`，负责人可直接在文档中修改，AI 下次读取时以人工修改为准。
- **当前落地**：在 `run_weekly_report.py` 的飞书通知卡片中增加纠偏提示文案，明确告知 VoidZ 可以通过直接回复机器人来补充本周遗漏的决策或纠正错误事实。
