# 活动规则

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/wiki/RdyRwEo20iu8DLkMq04jc57EpMe  
> **归属模块**: `mod_activity`  

## 业务目标

通过阶梯式优惠券绑定和充值加赠活动，提升用户充值金额和活跃度。

## 功能详情

### 1. 券码 (FP-ACTIVITY-001)

**功能描述**: 用户输入优惠码获取充值加赠优惠，系统实时校验优惠码及充值金额。

**业务规则**:
- 校验成功时展示优惠内容及预估奖励金额
- 充值金额未达标时提示最低起充额度要求
- 优惠券使用需按顺序完成，防止跳关
- 一次性优惠码使用后不可重复使用
- 无效优惠码提示（过期、拼写错误等）
- 活动结束后未使用的加赠礼包失效，充值页面恢复常规模式

### 2. 活动宣传图 (FP-ACTIVITY-002)

**功能描述**: 在充值页面展示活动宣传图，说明阶梯解锁及奖励发放逻辑。

**业务规则**:
- 用户必须按顺序完成阶梯解锁
- 每级有不同最低充值金额要求
- 加赠比例随轮次变化，后台动态配置
- 设有最高充值金额封顶，按公式计算奖励

### 3. 活动列表 (FP-ACTIVITY-003)

**功能描述**: 展示活动基本信息及状态，支持排序和交互体验优化。

**业务规则**:
- 活动状态优先排序：进行中 > 未开始 > 已结束
- 同状态下按优先级数值排序，优先级高靠前
- 优先级相同时按开始时间排序，越新越靠前
- 支持鼠标悬停浮起及图片放大效果
- 采用无限滚动加载活动列表
- 无有效活动时展示空状态文案

### 4. 活动详情页 (FP-ACTIVITY-004)

**功能描述**: 展示活动具体规则、条款及奖励发放时间等详细信息。

### 5. bonus卡片 (FP-ACTIVITY-005)

**功能描述**: 展示奖励提示及流水进度，支持提现确认弹窗交互。

**业务规则**:
- 点击或悬停显示奖励使用规则和流水计算规则
- 提现金额超过最大派彩金额时弹出二次确认弹窗

### 6. 首页banner (FP-ACTIVITY-006)

**功能描述**: 展示适配不同屏幕比例的banner，点击跳转至首充活动详情页。

**业务规则**:
- 支持PC端和移动端不同屏幕比例适配
- 点击banner跳转至首充活动详情页
