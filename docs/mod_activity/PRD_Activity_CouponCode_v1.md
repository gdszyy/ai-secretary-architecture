# 活动规则

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/wiki/WJzZwxHV8ijtWEkcwTAjEMJ0pWe  
> **归属模块**: `mod_activity`  

## 业务目标

通过优惠券码和阶梯加赠活动提升用户充值转化率和活跃度

## 功能详情

### 1. 券码 (FP-ACTIVITY-001)

**功能描述**: 用户输入优惠码获取充值加赠优惠，系统实时校验优惠码及充值金额

**业务规则**:
- 校验成功时展示优惠内容及预估奖励金额
- 充值金额未达标时提示最低起充额度要求
- 轮次/阶梯冲突时强制执行串行逻辑，防止跳关使用优惠码
- 一次性优惠码已使用时提示已使用
- 无效优惠码提示包括过期、未开始或不存在
- 活动时间结束后未使用的加赠礼包失效，充值页面恢复常规模式

### 2. 活动宣传图 (FP-ACTIVITY-002)

**功能描述**: 在充值页面展示活动宣传图，说明阶梯解锁及奖励发放逻辑

**业务规则**:
- 用户必须按顺序完成阶梯加赠券的使用
- 每级有不同的最低充值金额要求，后台动态配置
- 加赠比例随轮次变化，后台动态配置
- 设有最高充值金额封顶，奖励按公式计算

### 3. 活动列表 (FP-ACTIVITY-003)

**功能描述**: 展示活动基本信息及状态，支持排序和交互体验优化

**业务规则**:
- 活动状态优先排序：进行中 > 未开始 > 已结束
- 同状态下按后台配置优先级数字排序，数字越大越靠前
- 优先级相同时按开始时间排序，越晚越靠前
- 支持鼠标悬停卡片浮起或图片放大效果
- 采用无限滚动加载，初始加载10-20个活动，下滑自动加载更多
- 无有效活动时展示空状态及“敬请期待”文案

### 4. 活动详情页 (FP-ACTIVITY-004)

**功能描述**: 展示活动具体规则及条款，提供跳转充值或投注的操作按钮

**业务规则**:
- 活动进行中显示“去投注”按钮，活动结束按钮置灰或隐藏
- 展示活动规则、条款及奖励发放时间等详情

### 5. bonus卡片 (FP-ACTIVITY-005)

**功能描述**: 展示奖励及流水进度提示，支持转现确认弹窗交互

**业务规则**:
- Rewards提示图标点击或悬停显示使用规则及最大派彩金额
- UsageAmount提示图标展示流水进度，点击或悬停显示流水计算规则
- 当可提现金额大于最大派彩金额时，点击交易按钮弹出二次确认弹窗

### 6. 首页banner (FP-ACTIVITY-006)

**功能描述**: 展示适配不同屏幕比例的banner，点击跳转至首充活动详情页

**业务规则**:
- PC端和移动端分别适配不同屏幕比例
- 点击banner跳转首充活动详情页
