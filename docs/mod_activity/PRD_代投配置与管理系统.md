# 代投配置与管理系统需求说明

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/wiki/VxtqwRP2KiUvW6k9KLMjnkPJpCg  
> **归属模块**: `mod_activity`  

## 业务目标

建立一套代投配置与管理系统，支持第三方代理推广并整合主流广告媒体的Pixel/Tracking工具，实现代投流程规范、广告追踪及数据归因。

## 功能详情

### 1. 渠道商配置 (FP-ACTIVITY-001)

**功能描述**: 建立渠道商并配置商户Token，支持渠道分析及归因分类。

**业务规则**:
- 新增渠道商后展示在列表
- 渠道商新增后不可删除
- 可根据查询条件查询渠道商

### 2. 渠道配置页面 (FP-ACTIVITY-002)

**功能描述**: 建立渠道并关联渠道商，生成推广渠道链结，实现用户注册归因。

**业务规则**:
- 配置渠道给渠道商使用
- 渠道链结可直接选择配置项生成
- 依据渠道配置进行投放读取与归因分析

### 3. 投放渠道链接 (FP-ACTIVITY-003)

**功能描述**: 建立渠道推广链结，链结与渠道号唯一且不可重复。

**业务规则**:
- 建立渠道链结完成后可使用
- 通过链结打开平台
- 用户注册后纳入渠道归因

### 4. API 对接 (FP-ACTIVITY-004)

**功能描述**: 通过API接入GA、FB、Tiktok、Kwai等广告平台，发送用户注册、充值、投注等事件，实现数据回传与归因。

**业务规则**:
- 依据投放API接入第三方广告平台
- 用户注册、充值、投注时发送对应事件
- 后台检查事件接收状况并记录归因

### 5. 归因记录 (FP-ACTIVITY-005)

**功能描述**: 记录用户通过代投链结进入平台后的注册归因，推送并统计报表数据。

**业务规则**:
- 用户通过代投链结进入平台
- 注册时记录用户归因信息
- 归因数据用于推广日报与财务结算报表

### 6. 帐号绑定渠道商 (FP-ACTIVITY-006)

**功能描述**: 提供帐号绑定渠道商功能，用户登录后查看对应渠道报表，角色分为管理者和一般使用者。

**业务规则**:
- 帐号绑定后登录展示推广日报
- 帐号角色分为管理者和一般使用者
- 一般使用者仅能查看推广日报

### 7. 推广日报 (FP-ACTIVITY-007)

**功能描述**: 提供渠道推广日报表，支持按时间、渠道名称、总代及日期区间查询。

**业务规则**:
- 报表依据渠道号每日结算
- 支持日、周、月加总模式查询
- 支持模糊查找渠道名称及筛选总代

### 8. 财务结算 (FP-ACTIVITY-008)

**功能描述**: 依据渠道号进行每日结算，支持多条件筛选展示财务结算报表。

**业务规则**:
- 报表依据渠道号每日结算
- 支持代投时间区间筛选
- 支持总代、渠道ID及媒体筛选
- 查询条件为AND关系，未选择则不加条件

### 9. 汇率表配置 (FP-ACTIVITY-009)

**功能描述**: 管理汇率表，配置平台主币别与USD兑换关系。

**业务规则**:
- 可管理汇率表
- 配置当前平台币别与USD的汇率
