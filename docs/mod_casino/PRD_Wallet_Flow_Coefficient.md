# 钱包有效流水系数

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/docx/BDkvdJos9oX19Dx93SYjZZa6pBE  
> **归属模块**: `mod_casino`  

## 业务目标

解决多币别配置问题及风控用户的钱包冻结机制，保障充提投行为的合规与安全

## 功能详情

### 1. 支付代码配置（新功能） (FP-CASINO-001)

**功能描述**: 灵活配置商户相关的充提渠道代码，支持多币别和不同类型的支付代码管理

**业务规则**:
- 支持 MerchantId、CurrencyId、Type、AccountType、BankCode 等字段配置
- Type 仅支持 Deposit 和 Withdraw 两种，Withdraw 类型需填写 AccountType 和 BankCode
- 不同币别（如巴西币、墨西哥币）对应不同的 AccountType 和 BankCode 选项
- 支持新增、查询、编辑、删除操作，查询支持多条件或查询

### 2. 充提配置（Merchant） (FP-CASINO-002)

**功能描述**: 调整充提相关字段配置，优化商户充提管理

**业务规则**:
- 删除部分字段以简化配置

### 3. 用户管理 (FP-CASINO-003)

**功能描述**: 管理用户冻结与钱包冻结状态，区分账户冻结与钱包冻结的权限限制

**业务规则**:
- 用户冻结时踢出登录且无法登录
- 钱包冻结时禁止充值、投注和划转 Bonus，提款需人工审核
- 用户解冻后恢复正常操作权限

### 4. 提款 Create (FP-CASINO-004)

**功能描述**: 风控提款单管理，钱包冻结用户提款需人工审核流程

**业务规则**:
- 钱包冻结用户提款订单不直接发送三方，建立内部订单
- 人工审核通过后，手动发送三方提款请求
- 三方发送失败时支持重新发送，需保留前单备注

### 5. 提现-银行卡 (FP-CASINO-005)

**功能描述**: 根据手机号展示不同的银行卡提现字段及支付代码配置

**业务规则**:
- 根据手机号区分巴西和墨西哥用户，展示对应的支付代码和字段
- 巴西手机号展示 PIX 相关字段，自动带入 CPF
- 墨西哥手机号展示 CLABE 相关字段，自动带入 CURP
- 部分字段自动带入，部分字段隐藏

### 6. 充值 (FP-CASINO-006)

**功能描述**: 根据手机号展示对应币别的充值渠道，支持用户选择充值方式

**业务规则**:
- 巴西支持 PIX 充值渠道
- 墨西哥支持 CLABE 和 OXXO 充值渠道
- 充值流程包括选择渠道、金额及随机路由匹配

### 7. 主页钱包展示 (FP-CASINO-007)

**功能描述**: 展示用户各钱包总额及明细，支持跳转至详细交易列表

**业务规则**:
- 点击钱包金额展示各钱包总额
- 点击明细跳转对应交易明细页面
- 支持 Bonus 划转提示及站内信通知

### 8. 钱包冻结用户 (FP-CASINO-008)

**功能描述**: 限制钱包冻结用户的充值、投注和提款行为，提款需满足流水要求并人工审核

**业务规则**:
- 充值和投注操作失败提示联系客服
- 提款需满足1倍流水要求，否则提示未达提款要求
- 提款订单建立内部冻结金额，不直接发送三方
- 提款订单在 CMS 的风控提款单中管理
