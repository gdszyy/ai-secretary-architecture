# 游戏商户后台管理与钱包流程说明

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/docx/MkdRdaiPRokDnWxbFzijIMsvpwe  
> **归属模块**: `mod_casino`  

## 业务目标

实现游戏商户的后台管理及游戏钱包的准确结算与校验

## 功能详情

### 1. 游戏商户管理 (FP-CASINO-001)

**功能描述**: 提供游戏商户的新增、修改、启用、停用及维护时间配置功能

**业务规则**:
- 新增/修改时必填栏位包括名称、商户、API Key (Token)、API 地址、开放国别、Status
- 非必填栏位包括Secret Key、Maintain Start、Maintain End、Notice
- 加密方式支持AES、RSA、MD5、SHA256
- 新增商户默认状态为不开启
- 启用状态决定前台是否展示该商户及其游戏
- 维护时间内前台不展示该商户及游戏

### 2. 游戏钱包对应流程 (FP-CASINO-002)

**功能描述**: 根据不同钱包类型处理用户投注及结算回调流程

**业务规则**:
- 单纯主钱包流程中，游戏商用查用钱馀额需大于提款最低额度
- 单纯Bonus钱包流程中，游戏商用查用钱馀额小于提款最低额度，用户使用Bonus投注
- 混合型钱包流程中，用户充值且游戏商用查用钱馀额小于提款最低额度
- 结算回调时需区分钱包类型（Cash或Bonus）
- 需检查用户平台钱包与三方钱包余额是否一致
