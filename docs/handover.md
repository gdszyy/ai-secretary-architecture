## 1. 项目背景与目标

本项目旨在为 **XPBET 全球站** 构建一套完整的“功能地图 -> 业务文档 -> 执行看板”三层管理体系，并补齐所有缺失的 C 端前端交互需求文档。

项目涉及两个核心 GitHub 仓库：
1. **功能地图管理仓库**: `gdszyy/project-management-ai-secretary`
   - 存储了项目的整体规划、多维表格数据结构设计、前端可视化架构设计以及飞书三层体系打通方案。
   - 核心数据源：`tasks/tsk-92d7c861-577/deliverables/feature_doc_mapping.md`（包含 114 个功能点的完整清单）。

2. **前端交互需求文档仓库**: `gdszyy/xpbet-frontend-requirements`
   - 存储了所有的 PRD 和 UX Spec 文档。
   - 采用**混合矩阵模式（Hybrid Matrix Model）**：PRD 层（Why+What）+ UX Spec 层（How）双向索引。
