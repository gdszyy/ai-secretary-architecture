# KYC 三方身份验证配置与流程

> **文档类型**: PRD  
> **来源**: https://kjpp4yydjn38.jp.larksuite.com/docx/HCDOdfxZdoYVckxJ4mdjfN06pmg  
> **归属模块**: `mod_riskcontrol`  

## 业务目标

为满足合规与风控要求，提供可配置的第三方 KYC 服务及前台动态展示，限制未完成验证用户的关键业务操作。

## 功能详情

### 1. KYC商户配置 (FP-RISKCONTROL-001)

**功能描述**: 后台展示并管理已配置的 KYC 商户信息，支持按国家和名称查找及排序。

**业务规则**:
- 展示当前已配置的 KYC
- 支持依国家查找
- 支持依 KYC Name 查找
- 排序依ID倒序，最近新增的展示在第一笔

### 2. 建立 KYC 商户信息 (FP-RISKCONTROL-002)

**功能描述**: 后台创建新的 KYC 商户配置，填写必填栏位后保存。

**业务规则**:
- 点击 Create 创建新配置
- 必填栏位未填入时，点击送出展示红字提示
- 填写完成后返回列表展示新增设定

### 3. 修改 KYC 商户信息 (FP-RISKCONTROL-003)

**功能描述**: 后台编辑已存在的 KYC 商户配置，支持全栏位修改。

**业务规则**:
- 点击 Edit 进入编辑
- 必填栏位未填入时，点击送出展示红字提示
- 根据操作人员修改内容保存

### 4. 删除 KYC 商户信息 (FP-RISKCONTROL-004)

**功能描述**: 后台删除指定的 KYC 商户配置。

**业务规则**:
- 点击 Del 触发删除操作
- 展示确认框确认删除
- 点击送出后删除该笔资料

### 5. 前台 KYC 流程 (FP-RISKCONTROL-005)

**功能描述**: 前台根据后台配置动态展示 KYC 验证流程，限制未完成验证用户的关键业务操作。

**业务规则**:
- 用户完成 KYC 验证前限制充值、提款、体育投注/线上游戏下注等操作
- 后台关闭三方 KYC 时，前台仅收集身份证号/证件号码并校验格式后放行
- 用户必须完成 KYC 验证才能进行关键业务操作
