# 周报组件库规范 (Weekly Report Component Library)

在执行周报构建的第二阶段（模块提炼与优先级划分）和第三阶段（组件选型与数据绑定）时，必须参考本规范。本规范定义了周报模块的分类、核心 UI 组件的选型标准以及与用户交互的确认要素。

---

## 1. 模块分类与呈现优先级

周报的呈现结构分为两大类：**首屏模块 (Hero Modules)** 和 **详情展示模块 (Detail Modules)**。这种分层设计旨在确保核心信息能够第一时间触达读者，同时提供足够的深度以供下钻分析。

### 1.1 首屏模块 (Hero Modules) — 优先级最高 (P0)

首屏模块必须在用户打开周报的第一眼呈现，用于快速传递项目的整体健康度和核心指标。这些模块是整个周报的"门面"，决定了读者对项目当前状态的第一印象。

| 模块名称 | 核心定义与作用 |
| :--- | :--- |
| **项目健康度诊断 (Health Score)** | 综合评分，通常包含增长、产品、商业化、风险等维度的雷达图或评分卡。该模块需要直观地反映项目的整体运行状况。 |
| **北极星指标归因 (North Star Attribution)** | 核心指标的当前值、环比/同比变化，以及导致该变化的主要因素拆解。这是评估项目是否在正确轨道上的关键。 |
| **北极星因子分解图 (North Star Factor Breakdown)** | 将北极星指标拆解为各驱动因子，展示每个因子的当期值和对总指标的贡献量。适用于需要说明"为什么变化"的场景。 |
| **KPI 监控与数据异常 (KPI Monitor & Anomalies)** | 关键业务指标的红绿灯状态，特别是突破阈值的异常告警。该模块用于快速识别需要立即关注的问题领域。 |
| **里程碑进度 (Milestone Progress)** | 当前版本的核心目标达成情况，以及下一阶段的规划。这有助于对齐团队目标和预期。 |
| **多维健康度雷达图 (Multi-Dimension Health Radar)** | 多业务线/多维度的健康分雷达图，适用于扩张期/成熟期多条业务线并行的项目。 |
| **业务阶段进度 (Business Stage Progress)** | PMF 信号雷达图 + 阶段里程碑进度条，适用于种子期/启动期项目，展示 PMF 验证进度。 |

### 1.2 详情展示模块 (Detail Modules) — 优先级中等 (P1/P2)

详情展示模块通常通过首屏模块的下钻 (Drill-down) 或金刚区 (Navigation Bar) 跳转访问，提供更详细的业务数据和运营情况。这些模块为首屏的高层次指标提供了必要的上下文和细节支撑。

| 模块名称 | 核心定义与作用 |
| :--- | :--- |
| **业务模块详情 (Module Details)** | 各子业务线的具体进展、核心数据和近期动作，通常以模块进度卡片形式呈现。 |
| **增长漏斗详情 (Growth Funnel Detail)** | 从曝光到激活的完整漏斗，展示各环节转化率和流失量，适用于增长/电商/订阅模块。 |
| **指标趋势矩阵 (Metric Trend Matrix)** | 多指标横向对比表格，每行一个维度，配合迷你折线图，适用于渠道效率对比、SKU 健康度等。 |
| **用户分层分布 (User Segmentation)** | 用户生命周期分层的气泡图或分布图，适用于游戏玩家分层、订阅用户分层、B2B 客户健康分分布。 |
| **创作者/供给方生态 (Ecosystem Health)** | 头腰尾分布柱状图 + 活跃度趋势，适用于内容社区和平台/双边市场。 |
| **单元经济模型 (Unit Economics)** | LTV/CAC 比率大字展示 + 分解明细，适用于扩张期/成熟期商业化模块。 |
| **团队构成与动态 (Team & Organization)** | 人员变动、招聘需求、团队绩效。 |
| **财务情况 (Financial Status)** | 收入结构、成本支出、ROI 分析、现金流消耗。 |
| **风险与问题登记册 (Risk & Issue Register)** | 当前面临的阻塞问题、潜在风险点及应对策略。 |
| **决策与待办事项 (Decisions & Todos)** | 需要高层决策的事项列表，以及各模块的待办任务追踪。 |

## 2. 首屏组件选型规范 (Hero Component Specifications)

### 2.1 评分卡组件 (Scorecard Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 项目健康度诊断、KPI 监控 |
| **数据需求** | 指标名称、当前值、目标值、环比变化率、状态（healthy / warning / danger）、迷你趋势数据（可选） |
| **呈现方法** | 大字号数值展示，配合红绿灯颜色标识和迷你趋势图 (Sparkline)。数值进场时建议使用数字滚动动效。 |
| **下钻关联** | 点击可跳转至该指标的历史趋势图或详细归因分析页面。 |
| **口径说明** | 每个指标旁必须支持 ⓘ 悬停展示口径说明（数据来源系统 + 统计时间窗口 + 口径定义）。 |

### 2.2 归因瀑布图/树状图组件 (Attribution Chart Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 北极星指标归因说明 |
| **数据需求** | 总指标变化量、各子维度的贡献量（正向/负向）、维度名称 |
| **呈现方法** | 瀑布图展示各因素对总指标增减的贡献。正向贡献使用绿色系，负向贡献使用红色系。起始值和终止值使用深色标识。 |
| **下钻关联** | 点击特定子维度，跳转至该业务模块的详情页，查看具体的运营动作或产品变更。 |
| **口径说明** | 必须标注各维度的数据口径和来源模块。 |

### 2.3 北极星因子分解图 (North Star Factor Breakdown Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 北极星因子分解（所有有北极星指标的业务） |
| **数据需求** | 北极星指标名称和当期值、各因子名称和当期值、各因子对总指标的贡献率、因子环比变化方向 |
| **呈现方法** | 树状分解图（顶部为北极星指标，向下展开各因子）或桑基图（展示各因子流量贡献）。因子节点颜色根据环比变化方向（上升/下降/持平）动态着色。 |
| **下钻关联** | 点击某个因子节点，跳转至该因子对应的业务模块详情。 |
| **口径说明** | 每个因子节点必须支持 ⓘ 悬停展示该因子的口径说明和数据来源。 |
| **数据洞察集成** | 若某因子数据来自数据洞察探针的"待确认"项，则该节点显示 ❓ 标识，提示用户数据待核实。 |

### 2.4 时间轴组件 (Timeline Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 里程碑进度 |
| **数据需求** | 节点名称、计划时间、实际时间、状态（completed / in_progress / delayed / planned）、关键交付物 |
| **呈现方法** | 垂直或水平时间轴，当前节点高亮显示。已完成节点使用绿色标识，延期节点使用红色标识，进行中节点使用蓝色脉冲动效。 |
| **下钻关联** | 点击节点查看详细的资源消耗、阻塞问题或具体的交付物文档。 |

### 2.5 异常告警列表组件 (Anomaly Alert List Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 数据异常监控、风险登记册 |
| **数据需求** | 异常级别（P0/P1/P2）、异常描述、发生时间、影响范围、当前处理状态 |
| **呈现方法** | 带颜色标识的列表，高优异常置顶并闪烁提示。不同级别的异常使用不同的颜色和图标区分。 |
| **下钻关联** | 点击查看详细的排查报告、处理工单或相关的责任人信息。 |

### 2.6 多维健康度雷达图 (Multi-Dimension Health Radar Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 项目健康度诊断（多业务线/多维度场景） |
| **数据需求** | 各维度名称（对应业务模块）、各维度健康分（0-100）、各维度目标分（可选）、环比变化 |
| **呈现方法** | 多维雷达图，每个维度对应一个业务模块。当前值使用实线填充，目标值使用虚线轮廓。各维度颜色与模块颜色系统保持一致。 |
| **下钻关联** | 点击某个维度轴，跳转至对应业务模块的详情页。 |
| **使用条件** | 仅在有 3 个及以上业务模块需要同时展示健康度时使用；单模块健康度使用评分卡组件。 |

### 2.7 业务阶段进度组件 (Business Stage Progress Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 业务阶段进度（种子期/启动期项目） |
| **数据需求** | 当前阶段名称、PMF 信号指标（NPS、有机增长占比、核心路径留存、付费意愿）及其当期值、阶段里程碑列表 |
| **呈现方法** | 上方为 PMF 信号雷达图（展示各 PMF 维度的达成程度），下方为阶段里程碑进度条（展示从种子期到成熟期的进度）。 |
| **下钻关联** | 点击 PMF 信号维度，跳转至对应的产品/增长模块详情。 |
| **使用条件** | 仅在种子期或启动期项目中使用；成长期及以后使用标准健康度评分卡。 |

## 3. 详情展示组件选型规范 (Detail Component Specifications)

### 3.1 模块进度卡片组件 (Module Progress Card Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 业务模块详情概览 |
| **数据需求** | 模块名称、负责人、整体进度百分比、近期核心动作（3 条以内）、关键指标摘要（2-3 个） |
| **呈现方法** | 网格布局的卡片，包含进度条和简短文本。进度条颜色可根据状态（正常/延期/风险）动态变化。 |
| **下钻关联** | 点击进入该模块的完整数据看板，查看更详细的子任务和指标。 |

### 3.2 漏斗转化组件 (Funnel Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 增长漏斗详情、交易漏斗、订阅漏斗、冷启动漏斗 |
| **数据需求** | 各漏斗层级名称（按顺序）、各层级用户数/事件数、各层级转化率、各层级环比变化 |
| **呈现方法** | 垂直漏斗图，每层显示绝对值和转化率。层级宽度按比例缩放，直观展示流失量。环比变化用箭头和颜色标识（上升绿/下降红）。 |
| **下钻关联** | 点击某个漏斗层级，展示该层级的用户行为分布或流失原因分析。 |
| **口径说明** | 必须标注漏斗各层级的统计口径（如"下单"是否包含未支付订单）。 |

### 3.3 指标趋势矩阵 (Metric Trend Matrix Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 渠道效率矩阵、SKU 健康度、多指标横向对比 |
| **数据需求** | 维度名称列表、各维度的当期值、环比变化、7日/30日趋势折线数据（可选） |
| **呈现方法** | 表格布局，每行一个维度，列包含：维度名、当期值、环比变化（箭头+颜色）、迷你折线图 (Sparkline)。支持按指标排序。 |
| **下钻关联** | 点击某行，展示该维度的详细历史趋势图和归因说明。 |

### 3.4 用户分层气泡图 (User Segmentation Bubble Chart Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 用户生命周期管理、玩家分层、B2B 客户健康分分布 |
| **数据需求** | 分层名称列表、各层级用户数（气泡大小）、各层级平均活跃度（X 轴）、各层级平均 ARPU/健康分（Y 轴）、环比变化 |
| **呈现方法** | 气泡图（X=活跃度，Y=付费价值/健康分，气泡大小=用户数）。各分层使用不同颜色，支持悬停展示详细数据。 |
| **下钻关联** | 点击某个气泡，展示该分层用户的典型行为路径和运营建议。 |

### 3.5 创作者/供给方生态图 (Ecosystem Health Chart Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 创作者生态（内容社区）、供需平衡（平台/双边市场） |
| **数据需求** | 头部/腰部/长尾分层数量、各层级人均产出（发布量/服务量）、各层级留存率、环比变化 |
| **呈现方法** | 分层柱状图（展示各层级数量分布）+ 活跃度趋势折线（叠加在柱状图上）。柱状图颜色按层级深浅区分。 |
| **下钻关联** | 点击某个层级柱，展示该层级的代表性创作者/供给方列表和详细数据。 |

### 3.6 单元经济模型卡片 (Unit Economics Card Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 单元经济模型（扩张期/成熟期商业化模块） |
| **数据需求** | LTV、CAC、LTV/CAC 比率、Payback Period、各分解项明细（LTV 分解：ARPU × 留存周期；CAC 分解：营销费用 / 新增用户） |
| **呈现方法** | 顶部大字展示 LTV/CAC 比率（配合健康阈值标识：>3 为绿色，1-3 为黄色，<1 为红色），下方折叠面板展示分解明细。 |
| **下钻关联** | 点击 LTV 或 CAC 分解项，跳转至对应的商业化或增长模块详情。 |
| **口径说明** | 必须标注 LTV 的计算周期（12个月/24个月/生命周期）和 CAC 的统计口径（是否含品牌投放）。 |

### 3.7 口径说明浮层 (Data Source Tooltip Component)

| 属性 | 说明 |
| :--- | :--- |
| **适用模块** | 所有数据组件（通用附加组件，可附加到任意组件的任意指标上） |
| **数据需求** | 指标口径文本、数据来源系统名称、统计时间窗口、口径风险标识（若数据洞察探针标记了该指标） |
| **呈现方法** | 每个指标旁的 ⓘ 图标，悬停展示浮层。若该指标被数据洞察探针标记为"口径待确认"，则图标变为 ⚠️ 并使用黄色标识。 |
| **下钻关联** | 无（纯信息展示组件）。 |
| **使用规则** | 所有首屏组件的核心指标必须附加此组件；详情展示组件的关键指标建议附加。 |

## 5. 第三阶段交互的 4 要素确认规范

在向用户确认组件选型时，必须明确询问并确认以下 4 个核心要素，以确保最终生成的周报网页符合用户预期：

| 确认要素 | 核心问题与示例 |
| :--- | :--- |
| **数据来源** | 该组件的数据从第一阶段提炼的哪个业务模块中获取？（例如：DAU 数据来源于"增长与获客模块"）。若数据洞察探针标记了口径风险，必须在此步骤再次确认。 |
| **呈现方法** | 确认组件的视觉形态是否符合预期？（例如：是否需要趋势图？是否需要红绿灯告警？是否需要特定的颜色映射？）。 |
| **回报内容** | 组件内具体需要展示哪些数据字段？（例如：除了当前值，是否还需要展示同比/环比变化率、目标达成率等？）。 |
| **下钻逻辑** | 用户点击该组件后，期望跳转到哪个关联模块查看更详细的信息？（例如：点击"商业化健康度"评分卡，跳转到"财务情况"模块）。 |
