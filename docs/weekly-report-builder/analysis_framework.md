# 业务模块分析框架 (Business Module Analysis Framework)

在执行周报构建的第一阶段（解析与拆解）时，必须使用本框架对用户提供的输入文档进行深度解析。本框架旨在将非结构化的业务信息转化为可被 AI 代理和组件化看板直接消费的结构化数据。

---

## 1. 核心业务与商业模式提取

首先，从全局视角提取项目的核心信息，建立业务理解的基线：

- **核心业务定位**：项目解决什么问题？服务什么人群？其核心价值主张是什么？
- **商业模式闭环**：项目如何获取流量？如何产生价值？如何实现变现？
- **业务阶段判断**：根据用户规模、收入水平和增长状态，判断当前所处阶段（种子期 / 启动期 / 成长期 / 扩张期 / 成熟期）。
- **北极星指标 (North Star Metric)**：当前阶段最核心的单一关注指标是什么？该指标的波动如何影响全局业务？

---

## 2. 通用业务模块（5 大基础模块）

将项目拆解为具体的业务模块，并为每个模块匹配预设的评估模式。以下 5 大模块适用于所有商业模式，但不同阶段的呈现权重不同（详见第 4 节）。

### 2.1 增长与获客模块 (Growth & Acquisition)
- **模块定义**：关注项目如何获取新用户、扩大市场份额以及提升品牌曝光度。
- **关注重点**：渠道转化率、获客成本 (CAC)、新用户数、激活率、自然新增占比。
- **评估模式**：漏斗分析模型（追踪从曝光到激活的各环节流失）、ROI 评估（衡量渠道投放的投入产出比）。
- **常见异常**：流量激增但转化率暴跌、某渠道 CAC 异常升高、自然新增断崖式下跌。

### 2.2 产品与活跃模块 (Product & Engagement)
- **模块定义**：关注用户在产品内的行为表现、核心功能的渗透情况以及整体的活跃度。
- **关注重点**：DAU/MAU、留存率 (次日/7日/30日)、核心功能渗透率、用户使用时长、互动频次。
- **评估模式**：同期群分析 (Cohort Analysis)（追踪不同批次用户的长期留存）、活跃度健康模型（评估用户粘性）。
- **常见异常**：新功能上线后留存未提升、核心路径流失率增加、高活用户占比下降。

### 2.3 商业化与变现模块 (Monetization)
- **模块定义**：关注项目如何将用户价值转化为商业收入，以及收入结构的健康度。
- **关注重点**：ARPU/ARPPU、付费渗透率、LTV、GMV、退款率、各商业化场景收入占比。
- **评估模式**：LTV/CAC 比例（评估商业模式的可持续性）、收入结构拆解（分析各业务线的营收贡献）。
- **常见异常**：大R用户流失、某类商品/服务付费率骤降、单一商业化场景收入占比过高（如游戏消耗占比超 90%）。

### 2.4 风险与合规模块 (Risk & Compliance)
- **模块定义**：关注项目在运营过程中面临的各类风险，包括用户体验风险、资金风险和系统稳定性风险。
- **关注重点**：客诉率、风控拦截率、系统可用性 (SLA)、资金安全指标 (如 RTP - Return to Player)。
- **评估模式**：阈值告警模型（设定安全红线，触发即告警）、红线管理。
- **常见异常**：突破安全阈值（如 RTP 跌破 80% 或超过 100%）、集中性客诉、系统核心接口可用性下降。

### 2.5 里程碑与项目管理 (Milestones & Project Management)
- **模块定义**：关注项目版本的迭代进度、关键目标的达成情况以及团队的执行效率。
- **关注重点**：版本进度、关键节点达成情况、资源消耗、阻塞风险、待决策事项。
- **评估模式**：预期 vs 实际对比（评估交付延期情况）、延期风险评估。
- **常见异常**：关键路径被阻塞、跨部门协作效率低下、需求频繁变更导致延期。

## 3. 商业模式特有模块（按模式匹配）

在完成通用模块映射后，必须根据第 1 节识别的商业模式，从以下特有模块中选取适用项追加到模块列表。特有模块的优先级由 `analysis_engine.py` 根据商业模式自动推荐，用户可在阶段一交互中调整。

### 3.1 广告变现 (Ad-Monetized) 特有模块

适用产品：资讯、短视频、社区、内容聚合平台。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **广告库存与填充率 (Ad Inventory & Fill Rate)** | 广告位数量、填充率、eCPM 趋势、收入归因（品牌/效果/开屏/信息流） | 填充率骤降（需求侧预算收缩）、eCPM 异常波动（节假日/大促前后） |
| **内容消费深度 (Content Consumption Depth)** | 人均消费时长、内容完播率、推荐算法效率（CTR/CVR）、内容多样性指数 | 完播率下降（内容质量问题）、推荐 CTR 走低（算法衰退信号） |
| **用户画像质量 (Audience Quality)** | 标签覆盖率、定向精准度、高价值广告受众占比 | 标签覆盖率下降（数据采集问题）、高价值受众流失 |

### 3.2 订阅/会员 (Subscription) 特有模块

适用产品：视频平台、音乐平台、工具类 SaaS（C 端）、知识付费。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **订阅漏斗与流失 (Subscription Funnel & Churn)** | 试用转付费率、月/年订阅比、主动取消率 vs 被动流失率（支付失败）、挽回率 | 主动取消率突增（权益感知下降）、被动流失率上升（支付体验问题） |
| **会员权益使用率 (Benefit Utilization)** | 核心权益渗透率（专属内容/折扣使用率）、权益感知价值评分 | 核心权益使用率低（权益设计与用户需求不匹配） |
| **续约预测 (Renewal Forecast)** | 下月到期用户数、预测续约率、高流失风险用户分层 | 大批用户集中到期且续约意愿低（需提前干预） |

### 3.3 交易/电商 (Commerce) 特有模块

适用产品：电商平台、二手交易、本地生活、团购。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **交易漏斗 (Transaction Funnel)** | 浏览→加购→下单→支付→完成的各环节转化率、弃购率 | 加购率高但下单率低（价格/物流信任问题）、支付成功率下降（支付通道故障） |
| **供给侧健康度 (Supply Health)** | SKU 数量、库存周转率、缺货率、商家/卖家活跃度 | 热销品持续缺货（供应链问题）、商家活跃度下降（平台政策影响） |
| **履约与售后 (Fulfillment & After-sales)** | 发货时效、物流异常率、退换货率、客诉解决时长 | 退货率突增（商品质量/描述不符）、客诉解决时长超标 |
| **营销活动效果 (Campaign Performance)** | 活动 GMV 贡献、优惠券核销率、活动 ROI | 活动 ROI 低于历史均值（补贴效率下降）、核销率过低（活动设计问题） |

### 3.4 游戏/娱乐 (Gaming & Entertainment) 特有模块

适用产品：手游、棋牌、体育博彩、娱乐直播。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **游戏经济系统健康度 (Game Economy Health)** | 虚拟货币产出/消耗比、通货膨胀指数、道具/皮肤销售结构 | 货币通胀（产出过多导致付费意愿下降）、单一道具销售占比过高（经济结构脆弱） |
| **赛事与活动运营 (Event & Tournament Operations)** | 活动参与率、赛事观看人数、活动期间付费提升 | 活动参与率低于预期（活动设计/奖励吸引力不足）、活动后留存无提升 |
| **玩家分层与生命周期 (Player Segmentation & Lifecycle)** | 新手/成长/成熟/流失玩家分布、各层级 ARPU 对比 | 新手转化率低（新手引导问题）、成熟玩家流失加速（内容消耗殆尽） |
| **赔率与风控 (Odds & Risk Management)**（博彩专属）| RTP 趋势、大额投注监控、赔率模型准确率 | RTP 突破安全阈值（赔率模型失效或异常操控）、大额投注集中（风险暴露） |

### 3.5 B2B SaaS 特有模块

适用产品：CRM、ERP、数据工具、协作平台（企业服务）。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **合同与 ARR 健康度 (Contract & ARR Health)** | 新签 ARR、扩展 ARR（Expansion）、流失 ARR（Churn ARR）、净收入留存率（NRR） | NRR < 100%（流失超过扩展，商业模式不健康）、大客户流失（集中度风险） |
| **客户成功与健康分 (Customer Success & Health Score)** | 客户健康分分布、高风险客户数、QBR 完成率、NPS | 高风险客户数增加（流失预警）、NPS 持续下降（产品/服务质量问题） |
| **产品使用深度 (Product Adoption Depth)** | 功能渗透率、DAU/Seat 比率、API 调用量、核心工作流完成率 | DAU/Seat 比率低（用户未真正使用，续约风险高）、核心工作流完成率下降 |

### 3.6 平台/双边市场 (Marketplace) 特有模块

适用产品：招聘、房产、出行、外卖、自由职业平台。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **供需平衡 (Supply-Demand Balance)** | 供给方活跃度（司机/房东/商家）、需求方活跃度、供需比、等待时长/匹配率 | 供需比失衡（供给过剩或需求暴增）、匹配率下降（算法或供给质量问题） |
| **平台信任与安全 (Platform Trust & Safety)** | 评价体系健康度、欺诈率、纠纷率、平台责任赔付金额 | 欺诈率突增（安全漏洞）、纠纷率上升（规则不清晰或执行不一致） |
| **网络效应指标 (Network Effects)** | 用户互动密度、跨侧交叉购买率、口碑传播系数（K-factor） | K-factor 持续低于 1（网络效应未形成）、跨侧交叉购买率下降 |

### 3.7 内容与社区 (Content & Community) 特有模块

适用产品：UGC 平台、直播、论坛、知识社区。

| 模块名称 | 关注重点 | 常见异常 |
| :--- | :--- | :--- |
| **创作者生态 (Creator Ecosystem)** | 活跃创作者数、人均发布频次、创作者留存率、头部/腰部/长尾分布 | 头部创作者流失（竞争平台挖角）、腰部创作者激励不足导致断层 |
| **内容质量与分发 (Content Quality & Distribution)** | 优质内容率、内容违规率、算法分发 vs 关注分发比 | 违规率突增（内容审核漏洞）、算法分发占比过高（关注关系弱化） |
| **社区互动与氛围 (Community Engagement & Atmosphere)** | 互动率（评论/点赞/转发）、有效讨论占比、社区健康度指数 | 互动率持续下降（社区氛围恶化）、有效讨论占比低（内容质量问题） |

## 4. 业务阶段 × 模块建议矩阵

在识别业务阶段后，必须根据以下矩阵调整模块的呈现权重，并向用户说明禁用或降权的原因。

### 4.1 阶段定义

| 阶段 | 定义 | 典型特征 |
| :--- | :--- | :--- |
| **种子期 (Seed)** | PMF 探索阶段 | 用户 < 1 万，收入可忽略，核心任务是验证需求 |
| **启动期 (Launch)** | 冷启动/MVP 上线 | 用户 1 万~10 万，开始跑通核心路径 |
| **成长期 (Growth)** | 规模化增长 | 用户 10 万+，增长是第一优先级 |
| **扩张期 (Scale)** | 商业化提速 | 用户规模稳定，重点转向变现效率和多元化 |
| **成熟期 (Mature)** | 防守与优化 | 增长放缓，重点是留存、效率和新增长曲线 |

### 4.2 通用模块适用性矩阵

| 模块 | 种子期 | 启动期 | 成长期 | 扩张期 | 成熟期 |
| :--- | :---: | :---: | :---: | :---: | :---: |
| 增长与获客 | ⚠️ 轻量 | ✅ 必备 | ✅ 核心 | ✅ 必备 | ⚠️ 降权 |
| 产品与活跃 | ✅ 核心 | ✅ 核心 | ✅ 必备 | ✅ 必备 | ✅ 核心 |
| 商业化与变现 | 🚫 禁用 | ⚠️ 轻量 | ⚠️ 轻量 | ✅ 核心 | ✅ 核心 |
| 风险与合规 | 🚫 禁用 | ⚠️ 轻量 | ✅ 必备 | ✅ 必备 | ✅ 核心 |
| 里程碑与项目管理 | ✅ 核心 | ✅ 核心 | ✅ 必备 | ✅ 必备 | ✅ 必备 |

图例：✅ 必备/核心（首屏展示）| ⚠️ 轻量（简化版本，放入详情区）| 🚫 禁用（建议不展示，原因见下）

### 4.3 禁用/降权原因说明

| 规则 | 原因 |
| :--- | :--- |
| 种子期禁用商业化 | 收入数据极少，呈现会误导决策者认为商业化已成熟；此阶段应聚焦 PMF 验证 |
| 种子期禁用风险合规 | 系统未规模化，风控数据无统计意义；过早关注会分散核心注意力 |
| 成熟期降权增长 | 增长边际效益递减，应将首屏空间让给留存、商业化效率指标 |
| 启动期轻量商业化 | 仅展示"是否有付费用户"和"首单转化率"，不展示 ARPU/LTV 等需要大样本的指标 |

### 4.4 各阶段强制推荐的特有模块

| 阶段 | 强制推荐模块 | 原因 |
| :--- | :--- | :--- |
| 种子期 | **PMF 信号追踪**（增长变体）| 核心任务是验证需求，需要 NPS、留存曲线斜率、有机增长占比等 PMF 信号 |
| 启动期 | **冷启动漏斗**（增长变体）| 需要精细追踪首次体验路径的各环节转化，识别最大流失节点 |
| 成长期 | **渠道效率矩阵**（增长变体）| 多渠道并行时需要横向对比 CAC 和 LTV/CAC，识别最优渠道 |
| 扩张期 | **单元经济模型**（商业化变体）| 需要验证规模化后的盈利模型是否成立（LTV/CAC > 3 为健康） |
| 成熟期 | **用户分层与生命周期**（产品变体）| 需要精细化运营，区分高价值/流失风险用户，制定差异化策略 |

## 5. 框架外洞察 (Out-of-Framework Insights)

在解析文档时，如果发现以下 3 种触发条件，**必须直接提醒用户，并带用户回复决策**：

1. **未覆盖的新模式**：文档中出现了上述模块无法涵盖的重要业务逻辑（例如：全新的跨界合作模式、突发的外部政策影响）。
2. **反常识数据**：数据表现违背了常规商业逻辑（例如：提价后转化率反而上升、核心功能下线后留存率提升），需要用户确认是否为数据错误或特殊策略。
3. **隐性跨模块关联**：发现跨模块的深层因果关系（例如：技术重构导致了某项商业化指标的波动、风控策略调整导致了活跃度下降），但文档未明确指出。

## 6. 数据洞察探针规范 (Data Insight Probe Specification)

在完成模块映射后、输出结果前，必须运行以下 3 类数据洞察探针。探针的详细实现见 `scripts/data_insight_probe.py`。

### 6.1 口径来源诊断 (Data Source Probe)

**触发时机**：阶段一解析文档后，在输出模块拆解之前。

对文档中出现的每个关键指标，检测以下 4 类口径风险：

| 风险类型 | 检测规则 | 示例 |
| :--- | :--- | :--- |
| **口径缺失** | 指标有值但未说明统计口径 | DAU 是否包含机器人账号？留存是否区分新老用户？ |
| **时间窗口模糊** | 指标未明确时间范围 | "本周"是自然周还是业务周？ |
| **分母不明确** | 比率类指标未说明分母定义 | 付费率的分母是 MAU 还是 DAU？ |
| **口径不一致** | 同一指标在文档不同位置使用了不同口径 | 前文 DAU 含灰度用户，后文 DAU 不含 |

### 6.2 北极星因子分解关联询问 (North Star Factor Decomposition Probe)

**触发时机**：识别到北极星指标后，在进行模块映射之前。

根据北极星指标类型，使用标准分解公式检测各因子数据的完整性：

| 指标类型 | 标准分解公式 |
| :--- | :--- |
| **用户规模类**（DAU/MAU）| = 新用户 + 回流用户 + 留存用户 − 流失用户 |
| **收入类**（GMV/Revenue）| = 用户数 × 付费率 × ARPPU |
| **活跃类**（使用时长/频次）| = 用户数 × 人均频次 × 单次时长 |
| **效率类**（转化率/留存率）| = 分子指标 / 分母指标（需分别追踪） |

对每个因子检测：文档中是否已包含该因子的数据？若缺失，向用户询问并标记为"待补充"。

### 6.3 跨模块因果链诊断 (Cross-Module Causality Probe)

**触发时机**：完成所有模块映射后，作为框架外洞察的增强版。

检测以下常见跨模块因果链模式：

| 因果链模式 | 检测信号 |
| :--- | :--- |
| 风险模块异常 → 商业化波动 | RTP/客诉指标异常 + 收入同期变化 |
| 版本发布 → 产品指标变化 | 里程碑模块有版本上线 + 产品模块有留存/活跃变化 |
| 渠道结构变化 → 用户质量变化 | 增长模块渠道占比变化 + 产品模块留存/付费率变化 |
| 价格调整 → 转化率变化 | 商业化模块有定价变动 + 增长模块有转化率变化 |

## 7. 第一阶段输出格式要求

向用户输出第一阶段的分析结果时，必须结构清晰，可直接被 AI 解析使用。输出必须包含以下内容：

1. **核心业务与商业模式总结**：用 1-2 段话精炼概括项目的核心定位、商业闭环和业务阶段判断。
2. **数据洞察探针报告**：先于模块拆解输出口径诊断和北极星因子分解结果，标记所有待确认项。
3. **业务模块拆解列表**：列出识别出的通用模块 + 商业模式特有模块，并明确每个模块的关注重点和当前状态。注明因阶段原因被禁用/降权的模块及原因。
4. **框架外洞察与异常提醒**：单独列出发现的框架外洞察（如有），并说明触发条件。
5. **明确的行动号召 (Call to Action)**：
   - 邀请用户补充信息，确认商业逻辑和业务阶段判断。
   - **必须额外要求用户确认里程碑模块的准确性和当前状态。**
   - 引导用户对数据口径待确认项进行补充。
   - 引导用户对框架外洞察进行决策。
