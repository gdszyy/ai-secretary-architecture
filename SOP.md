# AI秘书标准作业程序 (SOP)

本SOP旨在指导项目经理如何高效地通过Telegram与AI秘书（Manus AI Agent）交互，以管理项目模块、记录进度、追踪问题和待办事项，并确保信息持久化到GitHub。

## 1. 交互总览

您在Telegram中发送的指令，AI秘书将解析并执行以下操作：
*   **信息解析**: 识别模块、操作类型、具体内容。
*   **GitHub操作**: 创建/更新Issue、添加评论、查询信息。
*   **定时提醒**: 对备忘事项设置提醒。

## 2. 核心指令格式

为了确保AI秘书能准确理解您的意图，请尽量使用清晰、包含关键信息的指令。

### 2.1 记录/更新模块信息

**目的**: 更新模块进度、记录问题、添加备注。

**格式**: `[模块名称]：[具体内容]`

**示例**: 
*   `基础数据-UOF：进度更新至80%，解决了数据对接问题，待办：与XX确认数据清洗方案。`
*   `支付系统：发现一个Bug，支付路由在特定情况下会失效，优先级高，请记录。`
*   `用户系统：今日已完成用户注册流程优化，待测试。`

**AI秘书行为**: AI秘书将在GitHub仓库中找到或创建对应的Issue，并将您的内容作为评论添加。如果内容涉及进度更新，可能会同步更新Issue标题或标签。

### 2.2 备忘与定时提醒

**目的**: 记录简短备忘，并设置提醒以确保后续完善。

**格式**: `[模块名称]：备忘，[备忘内容]`

**示例**: 
*   `支付系统：备忘，明天跟进EPAY支付路由配置。`
*   `活动中心：备忘，今日内需完善活动礼券预算配置细节。`

**AI秘书行为**: 
1.  在对应的GitHub Issue中添加一条评论，标记为“待完善备忘”。
2.  设置一个定时任务，在指定时间（或当日结束前）检查该备忘是否已完善。
3.  如未完善，通过Telegram发送提醒。

**如何标记备忘已完善**: 
*   **格式**: `[模块名称]备忘已完善，[补充说明]` 或 `[模块名称]备忘已完成。`
*   **示例**: 
    *   `支付系统备忘已完善，已与EPAY确认路由配置。`
    *   `活动中心备忘已完成。`
*   **AI秘书行为**: 更新GitHub Issue中的评论状态，并取消相关提醒。

### 2.3 查询模块信息

**目的**: 快速了解某个模块的最新状态、历史记录或遗留问题。

**格式**: `查询[模块名称][信息类型]`

**示例**: 
*   `查询用户系统最新进度和遗留问题。`
*   `告诉我基础数据-UOF模块的历史记录。`
*   `支付系统有什么待办事项？`

**AI秘书行为**: AI秘书将查询GitHub仓库中对应模块的Issue，整理其标题、标签、最新评论以及未关闭的待办事项，并通过Telegram回复给您。

## 3. GitHub Issue 管理约定

*   **Issue 标题**: 简洁明了，包含模块名称和关键状态。
*   **Labels**: 充分利用标签进行分类（如 `Module:XXX`, `Priority:高`, `Status:进行中`, `Type:Bug`, `Type:待办`, `Type:备忘`）。
*   **Comments**: 所有交互内容都将作为评论记录，确保历史可追溯。

## 4. 初始数据导入

如果您需要将现有Excel表格中的模块数据导入到GitHub Issue中，请直接在Telegram中告知AI秘书：

*   `请帮我把之前上传的 Excel 模块表导入到 GitHub 仓库。`

AI秘书将自动解析表格内容，并为每个模块创建对应的GitHub Issue。

## 5. 注意事项

*   **清晰沟通**: 您的指令越清晰，AI秘书的理解和执行就越准确。
*   **权限管理**: 确保Manus AI Agent拥有访问您GitHub仓库的必要权限。
*   **定期检查**: 尽管有AI秘书辅助，定期查看GitHub仓库也能帮助您更好地掌握项目全貌。

如有任何疑问或需要进一步的帮助，请随时向AI秘书提问。
