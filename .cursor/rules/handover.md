# handover 模块规范

## 模块定位

`docs/handover/` 目录存放**任务交接文档**，记录跨 Agent / 跨任务的上下文传递信息，确保后续接手的 Agent 或人员能快速恢复现场。

## 交接文档规范

每份交接文档必须包含以下结构：

1. **当前进度与现状** — 已完成什么、缺失什么
2. **核心设计决策** — 关键架构选型及原因
3. **下一步行动** — 接手 Agent 应立即执行的具体步骤
4. **相关文件索引** — 涉及的代码文件、设计文档、环境变量

## 禁止行为

- 禁止在交接文档中包含敏感凭证（App Secret、Token 等）
- 禁止交接文档超过 200 行，过长时应拆分为设计文档 + 交接摘要
